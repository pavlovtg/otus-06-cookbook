# REPORT

## AI-инструменты

### Cline + Claude Sonnet 4

Cline — AI-агент в VS Code на базе модели Claude Sonnet 4 (Anthropic). Использовался для генерации и редактирования документации, создания архитектурных артефактов (ADR, стандарты, AR), написания промтов и workflows. Работает в режиме Plan/Act с доступом к файловой системе и CLI.

### OpenRouter

OpenRouter — прокси-сервис для доступа к языковым моделям. Использовался как провайдер для Cline: позволил подключить Claude Sonnet 4 без прямой подписки на Anthropic API.

### OpenSpec с расширенной схемой ADR

OpenSpec — фреймворк для spec-driven разработки. Использовалась модифицированная схема `spec-driven-with-adr`, расширяющая стандартный OpenSpec: каждый change включает ADR-артефакт для фиксации архитектурных решений наряду с proposal, design, spec и tasks.

## Проблемы и решения

### Выбор структуры хранения архитектурных требований (2026-06-06)

Потребовалось значительное время на поиск подхода, совмещающего несколько требований одновременно: использование OpenSpec как фреймворка для spec-driven разработки, хранение архитектурных решений в ADR и стандартах, минимизация потребления токенов агентом, единая точка входа через `ARCHITECTURE.md`.
Решение: установлено расширение OpenSpec, написаны новые скилы для работы с архитектурой (`adr`, `ar`, `update-architecture-from-adr`, `update-architecture-from-ar`, `update-architecture-from-standard`), доработаны скилы OpenSpec (`openspec-propose`, `openspec-apply-change` и др.) под схему `spec-driven-with-adr`. Итоговая эффективность подхода будет проверена в ходе разработки.
