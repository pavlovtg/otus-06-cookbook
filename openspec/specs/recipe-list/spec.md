## Requirements

### Requirement: Список рецептов поддерживает фильтрацию и сортировку

Эндпоинт списка рецептов SHALL принимать дополнительные параметры `q` (строка поиска) и `sort` (порядок сортировки).
Параметры опциональны; при их отсутствии поведение не меняется по сравнению с предыдущей версией.
Параметры `q` и `sort` применяются совместно с существующими параметрами пагинации `page` и `pageSize`.

#### Scenario: Запрос без новых параметров

- **WHEN** клиент вызывает эндпоинт без параметров `q` и `sort`
- **THEN** система возвращает все рецепты с пагинацией, как прежде

#### Scenario: Запрос с параметром поиска

- **WHEN** клиент передаёт параметр `q`
- **THEN** система возвращает только рецепты, соответствующие поисковому запросу, с корректными значениями `total`, `page`, `pageSize`

#### Scenario: Запрос с параметром сортировки

- **WHEN** клиент передаёт параметр `sort`
- **THEN** система возвращает рецепты в указанном порядке сортировки

#### Scenario: Комбинация поиска, сортировки и пагинации

- **WHEN** клиент передаёт параметры `q`, `sort`, `page` и `pageSize`
- **THEN** система возвращает отфильтрованные и отсортированные рецепты с корректной пагинацией

### Requirement: Отображение списка рецептов в UI

Система SHALL отображать список рецептов на главной странице приложения. Каждая карточка рецепта MUST показывать название, описание, время приготовления, сложность и имя автора. Карточки MUST соответствовать макету из `docs/design/mockup/` (тёмная тема, градиентные плейсхолдеры фото, метаданные с иконками). Список SHALL отображаться постранично по 18 рецептов. Приватные рецепты отображаются только их автору и администратору.

#### Scenario: Загрузка страницы со списком

- **WHEN** пользователь открывает главную страницу приложения
- **THEN** отображается список рецептов с карточками, содержащими название, описание, время приготовления, сложность и имя автора

#### Scenario: Нет рецептов

- **WHEN** в системе нет рецептов
- **THEN** отображается сообщение об отсутствии рецептов

#### Scenario: Переход на детальную страницу

- **WHEN** пользователь кликает на карточку рецепта
- **THEN** пользователь переходит на детальную страницу этого рецепта

#### Scenario: Приватный рецепт виден только автору

- **WHEN** авторизованный пользователь открывает список рецептов
- **THEN** в списке отображаются публичные рецепты и приватные рецепты только этого пользователя

### Requirement: Наличие демо-данных при старте

Система SHALL содержать seed-данные: не менее 5 рецептов, загружаемых при первом запуске.

#### Scenario: Первый запуск приложения

- **WHEN** приложение запускается в чистом окружении
- **THEN** в системе присутствуют seed-рецепты и они отображаются в UI

### Requirement: Отображение фото на карточке рецепта

Карточка рецепта в списке SHALL отображать фото рецепта. Если фото не загружено — отображается заглушка.

#### Scenario: Карточка рецепта с фото

- **WHEN** пользователь открывает список рецептов
- **THEN** карточки рецептов с фото отображают загруженное фото

#### Scenario: Карточка рецепта без фото

- **WHEN** пользователь открывает список рецептов, и у рецепта нет фото
- **THEN** карточка отображает заглушку вместо фото
