### Requirement: Пагинация списка ингредиентов

Система SHALL поддерживать постраничную выдачу ингредиентов через query-параметры `page` и `pageSize`.
Ответ SHALL содержать поля `items`, `total`, `page`, `pageSize`.
Если параметры не заданы, система SHALL вернуть первую страницу с дефолтным размером 100.
`pageSize` SHALL быть ограничен максимумом 1000; если передано значение больше 1000, система SHALL использовать 1000 (не ошибка).
`page` и `pageSize` SHALL быть положительными целыми числами (≥ 1); иначе система SHALL вернуть `400`.

#### Scenario: Запрос без параметров пагинации

- **WHEN** клиент вызывает `GET /api/v1/ingredients` без параметров `page` и `pageSize`
- **THEN** система возвращает `200` с первой страницей, `pageSize = 100`, `page = 1`

#### Scenario: Запрос с явными параметрами пагинации

- **WHEN** клиент вызывает `GET /api/v1/ingredients?page=2&pageSize=50`
- **THEN** система возвращает `200` с элементами второй страницы, `page = 2`, `pageSize = 50`

#### Scenario: pageSize больше максимума

- **WHEN** клиент вызывает `GET /api/v1/ingredients?pageSize=5000`
- **THEN** система возвращает `200`, используя `pageSize = 1000`

#### Scenario: page равен нулю

- **WHEN** клиент вызывает `GET /api/v1/ingredients?page=0`
- **THEN** система возвращает `400`

#### Scenario: pageSize равен нулю

- **WHEN** клиент вызывает `GET /api/v1/ingredients?pageSize=0`
- **THEN** система возвращает `400`

#### Scenario: page отрицательный

- **WHEN** клиент вызывает `GET /api/v1/ingredients?page=-1`
- **THEN** система возвращает `400`

#### Scenario: Последняя страница содержит меньше элементов

- **WHEN** в базе 150 ингредиентов и клиент запрашивает `page=2&pageSize=100`
- **THEN** система возвращает `200` с 50 элементами, `total = 150`, `page = 2`, `pageSize = 100`

#### Scenario: Страница за пределами данных

- **WHEN** клиент запрашивает страницу, превышающую количество данных
- **THEN** система возвращает `200` с пустым `items` и корректным `total`
