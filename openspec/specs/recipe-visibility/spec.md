## Requirements

### Requirement: Выбор видимости рецепта при создании и редактировании

Система SHALL предоставлять переключатель «Публичный / Приватный» в форме создания и редактирования рецепта. Поле обязательное — пользователь явно выбирает видимость. По умолчанию предлагается «Публичный».

#### Scenario: Создание публичного рецепта

- **WHEN** авторизованный пользователь создаёт рецепт с видимостью «Публичный»
- **THEN** рецепт сохраняется как публичный и доступен всем пользователям

#### Scenario: Создание приватного рецепта

- **WHEN** авторизованный пользователь создаёт рецепт с видимостью «Приватный»
- **THEN** рецепт сохраняется как приватный и виден только автору и администратору

#### Scenario: Изменение видимости при редактировании

- **WHEN** автор рецепта меняет видимость с «Публичный» на «Приватный» и сохраняет
- **THEN** рецепт становится приватным и исчезает из списка для посторонних пользователей

### Requirement: Фильтрация приватных рецептов в API

Система SHALL скрывать приватные рецепты от неавторизованных пользователей и авторизованных пользователей, не являющихся автором. Администратор SHALL видеть все рецепты независимо от видимости.

#### Scenario: Неавторизованный пользователь запрашивает список рецептов

- **WHEN** неавторизованный пользователь запрашивает список рецептов
- **THEN** в ответе присутствуют только публичные рецепты

#### Scenario: Авторизованный пользователь запрашивает список рецептов

- **WHEN** авторизованный пользователь запрашивает список рецептов
- **THEN** в ответе присутствуют публичные рецепты и приватные рецепты этого пользователя

#### Scenario: Попытка получить чужой приватный рецепт

- **WHEN** пользователь запрашивает детальную страницу чужого приватного рецепта
- **THEN** система возвращает `403 Forbidden`

#### Scenario: Администратор видит все рецепты

- **WHEN** администратор запрашивает список рецептов
- **THEN** в ответе присутствуют все рецепты, включая приватные любых пользователей
