## Requirements

### Requirement: Детальная страница рецепта

Детальная страница SHALL отображать полную информацию о рецепте, включая категории в виде тегов, имя автора и признак публичности. Блок ингредиентов SHALL содержать интерактивный контрол масштабирования порций. Приватный рецепт SHALL быть доступен только автору и администратору. Детальная страница SHALL отображать секцию комментариев со списком всех комментариев к рецепту. Авторизованный пользователь, не оставивший комментарий, SHALL видеть форму добавления комментария. Авторизованный пользователь, уже оставивший комментарий, SHALL видеть сообщение об этом вместо формы. Неавторизованный пользователь SHALL не видеть форму добавления комментария.

#### Scenario: Открытие детальной страницы

- **WHEN** пользователь переходит по ссылке на рецепт
- **THEN** отображается детальная страница с полными данными рецепта, включая имя автора и признак публичности

#### Scenario: Рецепт не найден

- **WHEN** пользователь открывает URL несуществующего рецепта
- **THEN** отображается страница с сообщением «Рецепт не найден»

#### Scenario: Чужой приватный рецепт недоступен

- **WHEN** пользователь открывает URL чужого приватного рецепта
- **THEN** система возвращает `403 Forbidden` и отображается страница с сообщением об ошибке доступа

#### Scenario: Секция комментариев отображается на детальной странице

- **WHEN** пользователь открывает детальную страницу рецепта
- **THEN** отображается секция комментариев со списком всех комментариев (имя автора, дата, текст)

#### Scenario: Форма добавления комментария для авторизованного пользователя без комментария

- **WHEN** авторизованный пользователь, не оставивший комментарий к рецепту, открывает детальную страницу
- **THEN** отображается форма добавления комментария

#### Scenario: Форма скрыта если пользователь уже оставил комментарий

- **WHEN** авторизованный пользователь, уже оставивший комментарий к рецепту, открывает детальную страницу
- **THEN** форма добавления комментария не отображается

#### Scenario: Неавторизованный пользователь не видит форму комментария

- **WHEN** неавторизованный пользователь открывает детальную страницу рецепта
- **THEN** форма добавления комментария не отображается

### Requirement: Навигация с детальной страницы

Система SHALL предоставлять кнопку «Назад» для возврата к списку рецептов.

#### Scenario: Возврат к списку

- **WHEN** пользователь нажимает «Назад» на детальной странице
- **THEN** пользователь возвращается на страницу списка рецептов

### Requirement: Отображение фото на детальной странице

Детальная страница рецепта SHALL отображать фото рецепта. Если фото не загружено — отображается заглушка. Автор рецепта SHALL видеть кнопки «Загрузить фото» (если фото нет) или «Заменить фото» и «Удалить фото» (если фото есть).

#### Scenario: Детальная страница с фото

- **WHEN** пользователь открывает детальную страницу рецепта с фото
- **THEN** фото отображается в верхней части страницы

#### Scenario: Детальная страница без фото

- **WHEN** пользователь открывает детальную страницу рецепта без фото
- **THEN** на месте фото отображается заглушка

#### Scenario: Управление фото автором

- **WHEN** автор рецепта открывает детальную страницу рецепта без фото
- **THEN** отображается кнопка «Загрузить фото»

#### Scenario: Управление фото при наличии фото

- **WHEN** автор рецепта открывает детальную страницу рецепта с фото
- **THEN** отображаются кнопки «Заменить фото» и «Удалить фото»

### Requirement: Отображение категорий на детальной странице рецепта

Детальная страница рецепта SHALL отображать все категории рецепта как теги.

#### Scenario: Отображение рецепта с категориями

- **WHEN** пользователь открывает детальную страницу рецепта с категориями
- **THEN** страница отображает категории как теги

#### Scenario: Отображение рецепта без категорий

- **WHEN** пользователь открывает детальную страницу рецепта без категорий
- **THEN** страница отображается без тегов категорий

### Requirement: Детальная страница рецепта показывает рейтинг

Детальная страница рецепта SHALL отображать поле `averageRating` (средний рейтинг).
Для авторизованного пользователя ответ SHALL также содержать поле `myRating` (личная оценка, если выставлена).
Поле `averageRating` SHALL быть `null` если рецепт не имеет оценок.
Поле `myRating` SHALL быть `null` если пользователь не оценивал рецепт или не авторизован.

#### Scenario: Детальная страница содержит поля рейтинга

- **WHEN** клиент запрашивает детальную страницу рецепта
- **THEN** ответ содержит поля `averageRating` и `myRating`

#### Scenario: Рецепт без оценок на детальной странице

- **WHEN** клиент запрашивает детальную страницу рецепта без оценок
- **THEN** поле `averageRating` равно `null`

#### Scenario: Поле myRating для авторизованного пользователя с оценкой

- **WHEN** авторизованный пользователь, оценивший рецепт, запрашивает детальную страницу
- **THEN** поле `myRating` содержит его оценку (1–5)
