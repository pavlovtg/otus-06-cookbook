### Requirement: Просмотр каталога ингредиентов

Система SHALL отображать список всех ингредиентов, доступный без авторизации.
Список SHALL возвращаться постранично; формат ответа SHALL содержать поля `items`, `total`, `page`, `pageSize`.

#### Scenario: Открытие страницы каталога

- **WHEN** пользователь открывает `/ingredients`
- **THEN** отображается первая страница ингредиентов (до 100), сгруппированных по категориям, и пагинатор если страниц больше одной

#### Scenario: Фильтрация по названию

- **WHEN** пользователь вводит текст в поле поиска
- **THEN** список фильтруется по вхождению строки в название ингредиента (без учёта регистра), пагинация сбрасывается на первую страницу

#### Scenario: Фильтрация по категории

- **WHEN** пользователь выбирает категорию из выпадающего списка
- **THEN** список показывает только ингредиенты выбранной категории, пагинация сбрасывается на первую страницу

#### Scenario: Пустой результат фильтрации

- **WHEN** ни один ингредиент не соответствует фильтрам
- **THEN** отображается сообщение «Не найдено»

#### Scenario: Отображение полей ингредиента

- **WHEN** список ингредиентов загружен
- **THEN** каждый ингредиент показывает: название, единицу измерения, дефолтное количество, признак системного

#### Scenario: Навигация по страницам

- **WHEN** пользователь нажимает на номер страницы в пагинаторе
- **THEN** загружается соответствующая страница ингредиентов
