## MODIFIED Requirements

### Requirement: Отображение списка рецептов в UI

Система SHALL отображать список рецептов на главной странице приложения. Каждая карточка рецепта MUST показывать название, описание, время приготовления и сложность. Карточки MUST соответствовать макету из `docs/design/mockup/` (тёмная тема, градиентные плейсхолдеры фото, метаданные с иконками). Список SHALL отображаться постранично по 18 рецептов.

#### Scenario: Загрузка страницы со списком

- **WHEN** пользователь открывает главную страницу приложения
- **THEN** отображается список рецептов с карточками, содержащими название, описание, время приготовления и сложность

#### Scenario: Нет рецептов

- **WHEN** в системе нет рецептов
- **THEN** отображается сообщение об отсутствии рецептов

#### Scenario: Переход на детальную страницу

- **WHEN** пользователь кликает на карточку рецепта
- **THEN** пользователь переходит на детальную страницу этого рецепта
