# recipes-paging

## Зачем

Список рецептов возвращается целиком без разбивки на страницы — при росте базы это замедляет загрузку и перегружает интерфейс. PRD требует пагинацию списков; на странице должно отображаться 18 рецептов.

## Что меняется

- Список рецептов разбивается на страницы по 18 элементов.
- Появляется навигация по страницам.
- Отображается общее количество рецептов.

## Возможности

### Новые возможности

- `recipe-list-paging`: пагинация списка рецептов — навигация по страницам и индикация общего количества.

### Изменяемые возможности

- `recipe-list`: изменяется контракт ответа — с плоского массива на постраничный результат.

## Влияние

- API рецептов (backend и контракт).
- BFF и UI страницы списка рецептов.
- Тесты API.
