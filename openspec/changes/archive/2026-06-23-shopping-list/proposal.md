# shopping-list

## Зачем

Пользователь составил план меню на неделю, но не может автоматически получить список продуктов — приходится вручную выписывать ингредиенты из каждого рецепта. Список покупок закрывает этот пробел: агрегирует ингредиенты из всех блюд плана, суммирует количества и позволяет скопировать или распечатать результат.

## Что меняется

- Новая страница «Список покупок» с таблицей ингредиентов, сгруппированных по категориям.
- Ингредиенты из разных блюд суммируются; количество пересчитывается с учётом порций из планировщика.
- Кнопка «Скопировать» — копирует список текстом в буфер обмена.
- Кнопка «Распечатать» — открывает диалог печати браузера.
- Ссылка «Список покупок» на странице планировщика меню.

## Возможности

### Новые возможности

- `shopping-list-get`: просмотр агрегированного списка покупок по текущему плану меню
- `shopping-list-copy`: копирование списка покупок в буфер обмена
- `shopping-list-print`: печать списка покупок

### Изменяемые возможности

_(нет)_

## Влияние

- Новый раздел приложения, доступный только авторизованным пользователям.
- Зависит от плана меню: если план пустой — список покупок пустой.
- Список обновляется автоматически при изменении плана меню.
