## ADDED Requirements

### Requirement: Просмотр списка покупок

Авторизованный пользователь SHALL видеть список покупок, сформированный автоматически из текущего плана меню.
Список агрегирует ингредиенты из всех блюд всех слотов плана: одинаковые ингредиенты суммируются, количество пересчитывается с учётом порций, заданных в слотах планировщика.
Список сгруппирован по категориям ингредиентов; внутри каждой группы ингредиенты отсортированы по алфавиту.
Неавторизованный пользователь SHALL NOT иметь доступа к списку покупок.

#### Scenario: Пользователь открывает список покупок с заполненным планом

- **WHEN** авторизованный пользователь открывает страницу списка покупок и его план меню содержит блюда
- **THEN** отображается таблица ингредиентов, сгруппированных по категориям, с суммарным количеством и единицей измерения для каждого ингредиента

#### Scenario: Ингредиенты из разных блюд суммируются

- **WHEN** в плане меню несколько блюд содержат один и тот же ингредиент
- **THEN** в списке покупок этот ингредиент отображается одной строкой с суммарным количеством

#### Scenario: Количество пересчитывается по порциям

- **WHEN** в слоте планировщика для рецепта задано количество порций, отличное от базового
- **THEN** количество ингредиентов в списке покупок пересчитывается пропорционально заданным порциям

#### Scenario: Пустой план — пустой список

- **WHEN** авторизованный пользователь открывает страницу списка покупок и его план меню пустой
- **THEN** отображается сообщение о том, что список покупок пуст, и ссылка на планировщик меню

#### Scenario: Неавторизованный пользователь не видит список покупок

- **WHEN** неавторизованный пользователь пытается открыть страницу списка покупок
- **THEN** пользователь перенаправляется на страницу входа
