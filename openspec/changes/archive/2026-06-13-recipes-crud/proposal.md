# recipes-crud

## Зачем

Текущий бэкенд поддерживает только чтение списка рецептов, а доменная модель содержит лишь название и описание. Для реализации MVP необходим полный CRUD с расширенной моделью (время приготовления, сложность, пошаговые инструкции) и валидацией на клиенте и сервере. Интерфейс приводится к утверждённым макетам (`docs/design/mockup/`).

## Что меняется

- Расширение модели рецепта: добавляются поля «время приготовления», «сложность», «количество порций», «пошаговые инструкции».
- Новые операции: создание, просмотр детальной страницы, редактирование, удаление рецепта.
- Валидация данных рецепта на клиенте и сервере.
- UI приводится к дизайну из `docs/design/mockup/`: тёмная тема, карточки рецептов с расширенными полями, формы создания/редактирования, детальная страница.

## Возможности

### Новые возможности

- `recipe-create`: создание рецепта через форму с валидацией
- `recipe-detail`: просмотр детальной страницы рецепта
- `recipe-edit`: редактирование существующего рецепта
- `recipe-delete`: удаление рецепта с подтверждением

### Изменяемые возможности

- `recipe-list`: карточки рецептов отображают расширенные поля (время приготовления, сложность) и приводятся к новым макетам

## Влияние

- Backend: доменная модель, API-контракт, база данных, тесты
- Frontend: схемы валидации, BFF-функции, страницы и компоненты UI
- OpenAPI-контракт `docs/contracts/cookbook/recipes.yaml`
