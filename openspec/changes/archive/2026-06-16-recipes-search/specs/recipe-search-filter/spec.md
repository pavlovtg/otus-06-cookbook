## ADDED Requirements

### Requirement: Поиск рецептов по строке запроса

Система SHALL поддерживать поиск рецептов по единой строке запроса.
Поиск ведётся одновременно по: названию рецепта, описанию, названиям присвоенных категорий, названиям ингредиентов.
Слова в строке запроса интерпретируются как AND: каждое слово должно найтись хотя бы в одном из перечисленных полей.
При отсутствии параметра поиска возвращаются все доступные рецепты.

#### Scenario: Поиск по одному слову

- **WHEN** клиент передаёт параметр `q=борщ`
- **THEN** система возвращает только рецепты, у которых слово «борщ» встречается в названии, описании, категории или ингредиенте

#### Scenario: Поиск по нескольким словам

- **WHEN** клиент передаёт параметр `q=курица рис`
- **THEN** система возвращает только рецепты, у которых присутствуют оба слова (каждое — хотя бы в одном из полей)

#### Scenario: Поиск без совпадений

- **WHEN** клиент передаёт параметр `q=несуществующийингредиент`
- **THEN** система возвращает пустой список с `total=0`

#### Scenario: Пустой запрос

- **WHEN** клиент не передаёт параметр `q` или передаёт пустую строку
- **THEN** система возвращает все доступные рецепты без фильтрации
