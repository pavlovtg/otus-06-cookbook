# ingredients-paging

## Зачем

Страница `/ingredients` загружает все ингредиенты без ограничений — при росте каталога это создаёт нагрузку на БД и замедляет UI. Поле поиска по `title` не имеет ограничения длины на уровне API и UI, хотя домен ограничивает `title` до 200 символов.

## Что меняется

- `GET /api/v1/ingredients` возвращает постраничный результат вместо плоского массива.
- Добавляются query-параметры `page` и `pageSize`; если не заданы — возвращается первая страница дефолтного размера (100).
- `pageSize` ограничен максимумом 1000; если передано больше — используется 1000 (не ошибка).
- `page` и `pageSize` должны быть положительными числами; иначе `400`.
- Параметр `title` в поиске ограничивается максимальной длиной 200 символов; запрос с более длинным значением возвращает `400`.
- Ограничение длины `title` применяется на всех уровнях: API, application, UI.
- Правила пагинации фиксируются как архитектурное правило (AR).

## Возможности

### Новые возможности

- `ingredient-list-paging`: пагинация списка ингредиентов (page/pageSize, постраничный результат)
- `ingredient-search-title-limit`: ограничение длины строки поиска по title (max 200 символов) на уровне API и UI

### Изменяемые возможности

- `ingredient-list`: изменяется формат ответа (массив → постраничный результат) и добавляются query-параметры пагинации
