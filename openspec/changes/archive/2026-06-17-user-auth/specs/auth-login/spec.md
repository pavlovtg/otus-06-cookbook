## ADDED Requirements

### Requirement: Вход в систему

Система SHALL позволять пользователю войти по email и паролю. После успешного входа пользователь получает сессию, которая сохраняется между запросами.

#### Scenario: Успешный вход

- **WHEN** пользователь вводит корректные email и пароль
- **THEN** пользователь входит в систему и получает активную сессию

#### Scenario: Неверные учётные данные

- **WHEN** пользователь вводит несуществующий email или неверный пароль
- **THEN** вход отклоняется с сообщением об ошибке

#### Scenario: Пустые поля

- **WHEN** пользователь отправляет форму входа с незаполненными полями
- **THEN** вход отклоняется с указанием причины
