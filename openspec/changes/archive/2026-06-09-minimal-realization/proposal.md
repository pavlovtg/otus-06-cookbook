# minimal-realization

## Зачем

Приложение не имеет ни одной строки рабочего кода. Нужна минимальная рабочая версия, которую можно запустить и увидеть список рецептов в браузере — без авторизации, без лишних фич. Это первый шаг к MVP, без которого невозможно итерировать дальше.

## Что меняется

- Появляется backend-сервис рецептов с API для получения списка
- Появляется frontend со страницей списка рецептов
- Появляется инфраструктура для запуска всего приложения одной командой
- Добавляются демо-данные (рецепты) для проверки работы

## Возможности

### Новые возможности

- `recipe-list`: Просмотр списка публичных рецептов (название, описание, время приготовления, сложность)
- `infrastructure-compose`: Запуск всего приложения одной командой `docker compose up`

### Изменяемые возможности

_Нет — репозиторий не содержит существующих specs._

## Влияние

- Создаются новые компоненты: backend-сервис, API Gateway, frontend, база данных
- Затрагивается вся инфраструктура проекта (с нуля)
