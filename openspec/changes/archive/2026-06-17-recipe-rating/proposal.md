# recipe-rating

## Зачем

Пользователи не могут оценивать рецепты и находить лучшие по рейтингу — ключевая функция «Книги рецептов» из PRD-0001 не реализована. Рейтинг необходим для сортировки списка рецептов по популярности и отображения качества блюда на карточке.

## Что меняется

- Авторизованный пользователь может выставить рецепту оценку 1–5 звёзд; повторная оценка заменяет предыдущую.
- На карточке и детальной странице рецепта отображается средний рейтинг (виден всем) и оценка текущего пользователя (только авторизованным).
- Список рецептов поддерживает сортировку по убыванию среднего рейтинга.
- Неавторизованный пользователь видит средний рейтинг, но не может ставить оценку.

## Возможности

### Новые возможности

- `recipe-rating-set`: авторизованный пользователь выставляет или обновляет оценку рецепту (1–5 звёзд)
- `recipe-rating-display`: средний рейтинг и оценка текущего пользователя отображаются на карточке и детальной странице

### Изменяемые возможности

- `recipe-list-sort`: добавляется значение `rating_desc` к существующим `title_asc` / `title_desc`
- `recipe-list`: ответ дополняется полями `averageRating` и `myRating` для каждого рецепта
- `recipe-detail`: ответ дополняется полями `averageRating` и `myRating`

## Влияние

- Backend: новая таблица `recipe_ratings`; новые эндпоинты для выставления и удаления оценки; расширение DTO рецептов.
- Frontend: компонент звёздного рейтинга на карточке и детальной странице; BFF-функции для работы с оценками.
- OpenAPI-контракт: новые операции и поля в `docs/contracts/cookbook/cookbook.yaml`.
- Seed-данные: добавить оценки для демо-рецептов.
