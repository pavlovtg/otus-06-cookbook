# dashboards

## Зачем

В приложении нет страницы со сводной статистикой: пользователь не видит ни общей картины по рецептам, ни своей активности, ни заполненности плана меню. Дашборд закрывает этот пробел и делает приложение информативным для всех ролей.

## Что меняется

- Новая страница «Дашборд», доступная всем пользователям.
- Набор виджетов зависит от роли: гость, авторизованный пользователь, администратор.
- KPI-карточки: общее количество рецептов, рецепты пользователя, комментарии, пользователи (для admin).
- Топ-10 рецептов по рейтингу — список/таблица.
- Топ-10 рецептов из избранного по рейтингу — только для авторизованных.
- Гистограммы: количество рецептов по категории «Основной ингредиент» и «Национальная кухня».
- Мини-сетка заполненности плана меню (пн–вс × завтрак/обед/ужин) — только для авторизованных.
- Для администратора вместо избранного и плана меню — топ-10 пользователей по рейтингам и комментариям.
- Новый пункт «Дашборд» в навигации (виден всем).

## Возможности

### Новые возможности

- `dashboard-stats`: получение статистики дашборда с учётом роли пользователя
- `dashboard-page`: страница дашборда с виджетами, зависящими от роли

### Изменяемые возможности

_(нет)_

## Влияние

- Зависит от существующих данных: рецепты, рейтинги, комментарии, избранное, план меню, пользователи.
- Навигация: добавить пункт «Дашборд».
