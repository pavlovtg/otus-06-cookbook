## ADDED Requirements

### Requirement: Статистика для гостя

Гость SHALL видеть: общее количество рецептов, топ-10 рецептов по рейтингу, распределение рецептов по основному ингредиенту, распределение рецептов по национальной кухне.

#### Scenario: Гость открывает дашборд

- **WHEN** неавторизованный пользователь открывает страницу дашборда
- **THEN** отображаются KPI «Всего рецептов», топ-10 по рейтингу и две гистограммы

### Requirement: Статистика для авторизованного пользователя

Авторизованный пользователь SHALL видеть: общее количество рецептов, количество своих рецептов, количество своих комментариев, топ-10 рецептов по рейтингу, топ-10 рецептов из своего избранного по рейтингу, заполненность своего плана меню на неделю.

#### Scenario: Авторизованный пользователь открывает дашборд

- **WHEN** авторизованный пользователь открывает страницу дашборда
- **THEN** отображаются три KPI, два топ-10 списка и мини-сетка плана меню

#### Scenario: Избранное пустое

- **WHEN** у пользователя нет рецептов в избранном
- **THEN** топ-10 из избранного отображается как пустой список

#### Scenario: План меню не заполнен

- **WHEN** у пользователя нет блюд в плане меню
- **THEN** все слоты мини-сетки отображаются как незаполненные

### Requirement: Статистика для администратора

Администратор SHALL видеть: общее количество рецептов, количество пользователей, общее количество комментариев, топ-10 рецептов по рейтингу, топ-10 пользователей по количеству выставленных рейтингов, топ-10 пользователей по количеству комментариев.

#### Scenario: Администратор открывает дашборд

- **WHEN** пользователь с ролью администратора открывает страницу дашборда
- **THEN** отображаются три KPI и три топ-10 списка

### Requirement: Топ-10 рецептов по рейтингу

Список топ-10 SHALL содержать не более 10 рецептов, отсортированных по среднему рейтингу по убыванию.

#### Scenario: Рецептов больше 10

- **WHEN** в системе более 10 рецептов с оценками
- **THEN** отображаются ровно 10 рецептов с наибольшим рейтингом

#### Scenario: Рецептов меньше 10

- **WHEN** в системе менее 10 рецептов
- **THEN** отображаются все доступные рецепты

### Requirement: Распределение рецептов по категориям

Гистограммы SHALL показывать количество рецептов для каждого значения категорий «Основной ингредиент» и «Национальная кухня». Категории без рецептов не отображаются.

#### Scenario: Есть рецепты с категориями

- **WHEN** рецепты имеют назначенные категории
- **THEN** гистограмма показывает столбец для каждой категории с хотя бы одним рецептом
