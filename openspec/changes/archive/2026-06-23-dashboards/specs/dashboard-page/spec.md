## ADDED Requirements

### Requirement: Страница дашборда доступна всем

Страница дашборда SHALL быть доступна без авторизации. Набор виджетов зависит от роли текущего пользователя.

#### Scenario: Гость переходит на дашборд

- **WHEN** неавторизованный пользователь открывает страницу дашборда
- **THEN** страница отображается с виджетами для гостя

#### Scenario: Авторизованный пользователь переходит на дашборд

- **WHEN** авторизованный пользователь открывает страницу дашборда
- **THEN** страница отображается с расширенным набором виджетов

### Requirement: Пункт «Дашборд» в навигации

Пункт «Дашборд» SHALL отображаться в главном меню для всех пользователей, включая гостей.

#### Scenario: Гость видит навигацию

- **WHEN** неавторизованный пользователь смотрит на главное меню
- **THEN** пункт «Дашборд» виден и доступен для перехода

### Requirement: KPI-карточки

Числовые метрики SHALL отображаться в виде KPI-карточек с подписью и значением.

#### Scenario: Отображение KPI

- **WHEN** страница дашборда загружена
- **THEN** каждая метрика показана в отдельной карточке с числом и подписью

### Requirement: Топ-10 рецептов — список с рейтингом

Топ-10 рецептов SHALL отображаться в виде пронумерованного списка с названием рецепта и средним рейтингом. Название является ссылкой на детальную страницу рецепта.

#### Scenario: Клик по названию рецепта в топ-10

- **WHEN** пользователь нажимает на название рецепта в топ-10
- **THEN** происходит переход на детальную страницу этого рецепта

### Requirement: Гистограммы распределения рецептов

Распределение рецептов по категориям SHALL отображаться в виде столбчатой диаграммы.

#### Scenario: Гистограмма с данными

- **WHEN** есть рецепты с категориями «Основной ингредиент» или «Национальная кухня»
- **THEN** гистограмма отображает столбцы для каждой категории

### Requirement: Мини-сетка плана меню

Заполненность плана меню SHALL отображаться в виде сетки 7 дней × 3 приёма пищи, где каждый слот визуально отмечен как заполненный или пустой.

#### Scenario: Частично заполненный план

- **WHEN** у пользователя заполнена часть слотов плана меню
- **THEN** заполненные слоты выделены, пустые — нет
