# recipe-photos

## Зачем

Рецепты в системе не имеют фотографий — карточки и детальная страница показывают только текстовые данные. Без фото пользователь не может визуально оценить блюдо, что снижает привлекательность приложения и не соответствует требованиям PRD.

## Что меняется

- Рецепт получает фотографию: пользователь может загрузить фото при создании или отдельно при редактировании.
- Фото отображается на карточке рецепта (в списке) и на детальной странице.
- Фото рецепта можно заменить или удалить — отдельными действиями, независимо от редактирования остальных полей.
- Seed-данные включают фотографии для демо-рецептов.
- Если фото не загружено — показывается заглушка.

## Возможности

### Новые возможности

- `recipe-photo-upload`: загрузка фото к рецепту с валидацией формата и размера
- `recipe-photo-delete`: удаление фото рецепта
- `recipe-photo-serve`: раздача фото и thumbnail через публичный endpoint

### Изменяемые возможности

- `recipe-detail`: добавляется отображение фото (или заглушки) и кнопки управления фото
- `recipe-create`: добавляется возможность загрузить фото при создании рецепта
- `recipe-edit`: добавляется возможность заменить или удалить фото при редактировании рецепта
- `recipe-list`: карточки рецептов отображают thumbnail (или заглушку)

## Влияние

- Модель данных рецепта (новое поле `photoId`).
- REST API рецептов (новые endpoints для загрузки, удаления и раздачи фото; расширение DTO).
- Backend: обработка изображений, хранение в БД.
- Frontend: схемы, BFF, UI карточки, детальной страницы и формы рецепта.
- OpenAPI контракт.
