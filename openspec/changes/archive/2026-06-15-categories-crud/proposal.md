# categories-crud

## Зачем

Рецепты не имеют категорий и тегов — пользователь не может фильтровать и классифицировать блюда по типу (завтрак, десерт, веган и т.д.). Справочник категорий необходим для поиска и навигации по рецептам и является требованием MVP.

## Что меняется

- Новая сущность `Category` с полями `id`, `name`, `description`, `type` (enum из 7 значений).
- CRUD API для категорий: список, создание, редактирование, удаление.
- Ограничение: удаление только если категория не используется ни в одном рецепте.
- Ограничение: `type` immutable — нельзя изменить после создания.
- Seed-данные: ~60 категорий по 7 типам согласно PRD.
- Страница `/categories`: список категорий, сгруппированных по типу, с CRUD-операциями (доступно всем, т.к. авторизация ещё не реализована).

## Возможности

### Новые возможности

- `category-list`: получение списка всех категорий
- `category-create`: создание категории
- `category-edit`: редактирование названия и описания категории (type immutable)
- `category-delete`: удаление категории (запрет если используется в рецептах)
