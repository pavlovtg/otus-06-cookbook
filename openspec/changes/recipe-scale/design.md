# recipe-scale

## Контекст

Детальная страница рецепта (`apps/web/app/recipes/[id]/page.tsx`) — Server Component. Блок ингредиентов отображает количества статически, без возможности изменения. Поле `servings` уже присутствует в `RecipeDto` и передаётся на страницу. Backend и API изменений не требуют.

## Цели / Не-цели

**Цели:**

- Добавить интерактивный контрол «+/−» для изменения числа порций.
- Пересчитывать количества ингредиентов на клиенте при изменении порций.
- Не трогать backend и API.

**Не-цели:**

- Сохранение выбранного числа порций между сессиями.
- Ввод числа порций вручную (только кнопки +/−).
- Изменение базового `servings` рецепта.

## Решения

### Вынос блока ингредиентов в Client Component

Текущая страница — Server Component. Интерактивность (useState) требует Client Component. Решение: вынести блок ингредиентов с контролом порций в отдельный `IngredientsCard` (`"use client"`). Страница остаётся Server Component и передаёт `ingredients` и `servings` как props.

Альтернатива — сделать всю страницу Client Component — отклонена: потеря SSR и лишние запросы к API с клиента.

### Формула пересчёта

`displayAmount = baseAmount × (currentServings / baseServings)`, округление до 2 знаков после запятой. Базовое значение `baseServings` фиксируется при монтировании компонента и не меняется.

### Ограничения диапазона

Минимум 1, максимум 99 — соответствует PRD и макету. Кнопка «−» блокируется при `currentServings === 1`, кнопка «+» — при `currentServings === 99`.

## Риски / Компромиссы

- [Дробные значения] При делении могут возникать длинные дроби → Mitigation: `Math.round(x * 100) / 100`.
- [Очень малые количества] При уменьшении порций до 1 некоторые ингредиенты могут показывать 0 → Mitigation: показывать минимум 0.01 или оставить как есть (приемлемо для MVP).
