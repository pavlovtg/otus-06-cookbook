# ingredients-crud

## Зачем

Приложение не имеет управления ингредиентами: нет каталога, нет возможности создавать, редактировать и удалять ингредиенты. Без этого невозможно реализовать добавление ингредиентов в рецепты. Ограничения по правам доступа и проверка использования в рецептах реализуются позже — после авторизации и ингредиентов в рецепте.

## Что меняется

- Новый агрегат `Ingredient` в bounded context `cookbook`: поля `id`, `title`, `unit`, `default_amount`, `category` (enum), `is_system` (bool).
- CRUD-эндпоинты ингредиентов: `GET /api/v1/ingredients`, `POST /api/v1/ingredients`, `GET /api/v1/ingredients/{id}`, `PUT /api/v1/ingredients/{id}`, `DELETE /api/v1/ingredients/{id}`.
- Фильтрация списка по названию и категории через query-параметры.
- Валидация на клиенте и сервере.
- Seed-данные: 50+ системных ингредиентов.
- UI: страница `/ingredients` по макету из `docs/design/mockup/`.
- OpenAPI-контракт `docs/contracts/cookbook/recipes.yaml` расширяется схемами и эндпоинтами ингредиентов.

## Возможности

### Новые возможности

- `ingredient-list`: просмотр каталога ингредиентов с фильтрацией по названию и категории
- `ingredient-create`: создание ингредиента через форму с валидацией
- `ingredient-edit`: редактирование существующего ингредиента
- `ingredient-delete`: удаление ингредиента (без проверки использования в рецептах — добавим позже)
