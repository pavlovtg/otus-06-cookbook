# recipe-comments

## Зачем

Рецепты не поддерживают комментарии — авторизованные пользователи не могут оставлять отзывы и делиться впечатлениями. Это требование MVP из PRD-0001.

## Что меняется

- Авторизованный пользователь может оставить один комментарий к рецепту (текст до 2000 символов).
- Автор комментария может удалить свой комментарий.
- Автор рецепта и администратор могут удалить любой комментарий к рецепту.
- Комментарии видны всем, включая неавторизованных пользователей.
- Форма добавления комментария скрыта, если пользователь уже оставил комментарий к данному рецепту.
- Детальная страница рецепта отображает список комментариев с именем автора и датой.
- Seed-данные: 20+ комментариев к рецептам.

## Возможности

### Новые возможности

- `recipe-comment-add`: авторизованный пользователь добавляет комментарий к рецепту (один на рецепт)
- `recipe-comment-delete`: удаление комментария автором комментария, автором рецепта или администратором
- `recipe-comment-list`: получение списка комментариев к рецепту (публично)

### Изменяемые возможности

- `recipe-detail`: добавляется секция комментариев на детальной странице рецепта

## Влияние

- Backend: новая таблица `recipe_comments`, новые эндпоинты для комментариев, миграция EF Core, seed-данные.
- OpenAPI-контракт `docs/contracts/cookbook/recipes.yaml`: добавляются новые пути для комментариев.
- Frontend: BFF-функции для комментариев, обновление детальной страницы рецепта.
- E2E-тесты: новый файл `tests/e2e/test_comments_api.py`.
