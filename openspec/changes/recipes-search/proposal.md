# recipes-search

## Зачем

Список рецептов не поддерживает поиск и фильтрацию: пользователь не может найти рецепт по названию, ингредиентам или категориям. Это делает приложение непригодным при росте числа рецептов и является базовым требованием PRD-0001 MVP.

## Что меняется

- Поиск по строке: одно текстовое поле ищет одновременно по названию, описанию, категориям и ингредиентам рецепта (логика AND по словам).
- Сортировка списка: по алфавиту или по рейтингу.
- Автодополнение: при вводе ≥ 2 символов показываются подсказки из категорий и ингредиентов.
- Состояние поиска и сортировки сохраняется в URL.

## Возможности

### Новые возможности

- `recipe-search-filter`: поиск рецептов по строке запроса (название, описание, категории, ингредиенты; AND-логика по словам)
- `recipe-list-sort`: сортировка списка рецептов по названию — по возрастанию (А→Я) или по убыванию (Я→А)
- `recipe-search-autocomplete`: автодополнение при вводе — подсказки из категорий и ингредиентов

### Изменяемые возможности

- `recipe-list`: добавляются параметры поиска и сортировки; поведение без параметров не меняется

## Влияние

- Backend: эндпоинт списка рецептов, сервис и репозиторий.
- Frontend: страница списка рецептов, BFF-слой.
- Тесты: e2e API и UI e2e.
- Миграций БД не требуется.
