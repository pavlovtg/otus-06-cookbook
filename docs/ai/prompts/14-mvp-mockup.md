# MVP-макет «Книга рецептов» в виде HTML

## Роль

Frontend-прототипировщик: создаёт кликабельный HTML-прототип MVP без реального бэкенда.

## Контекст

- Требования: `docs/prd/PRD-0001-MVP.md`
- Уточнённые требования: `docs/prd/PRD-0001-MVP-clarification.md`
- Визуальный стиль: `docs/design/guide/STYLE_GUIDE.md`
- Backend не реализован — нужны мок-данные и имитация REST-вызовов.
- Макет передаётся в разработку и используется как основа для Storybook в Next.js.

## Подготовка

Перед началом работы прочитать `docs/design/guide/STYLE_GUIDE.md` и извлечь design tokens:

- Цвета (палитра bg/fg/accent/data)
- Шрифт, веса, размеры, line-height, letter-spacing
- Радиусы
- Тени (box-shadow: inset rim + outer)
- Отступы (padding карточек, gap сетки, вертикальные отступы между секциями)

Все извлечённые токены оформить как CSS-переменные в `styles.css` и использовать их во всей вёрстке.

## Задача

- Собрать HTML-прототип в папке `docs/design/mockup/`, покрывающий все пользовательские сценарии MVP.
- Реализовать переходы между экранами внутри одного `index.html` (hash-роутинг или show/hide секций).
- Все данные — моки в JS; сетевые вызовы — через обёртку `fakeApi.*`, возвращающую `Promise` с `setTimeout` (имитация задержки 150–400 мс).
- Использовать skill `emil-design-eng` для создания дизайна.

## Артефакты

```
docs/design/mockup/
├── index.html      # точка входа, роутер, JS-логика
└── styles.css      # CSS-переменные из STYLE_GUIDE + все стили
```

## Технические требования

- `index.html`: HTML + inline JS, без npm/сборки.
- `styles.css`: подключается через `<link>`, содержит CSS-переменные токенов и все стили.
- Внешние зависимости только через CDN: Google Fonts (Inter), Chart.js (для дашборда).
- Язык интерфейса — русский.
- Адаптивная вёрстка: desktop (≥1280px) + mobile (≤768px).
- Валидация форм на клиенте (required, типы, длина).
- Все состояния UI реализованы: empty, loading, error, success.

## Покрытие функциональности MVP

### Авторизация

- Формы логина и регистрации.
- Три предзаполненных пользователя: `user1`, `user2`, `admin` (роль `admin`).
- Имитация JWT (токен в `localStorage`); выход из системы.
- При `401` — модальное окно входа или редирект на страницу логина.
- Элементы управления, недоступные текущей роли, скрываются (не блокируются).

### Рецепты

- Список с пагинацией (100 элементов на страницу).
- Карточка: фото (первое из галереи или заглушка), название, категории-теги, время, сложность, имя автора, общий рейтинг, мой рейтинг (если оценил), кнопка «В избранное» (только авторизованным), тег «Приватный» (если `is_public = false`).
- Клик по карточке → детальная страница (без промежуточного попапа).
- Детальная страница: галерея фото, заголовок, сложность, время, автор, категории, признак публичности, описание, ингредиенты с пересчётом порций, пошаговые инструкции, рейтинг (inline, 1–5 звёзд), комментарии.
- Органы управления на детальной странице: «Назад» (с сохранением позиции прокрутки и поискового запроса), «В избранное», «Задать рейтинг», «Пересчитать порции», «Редактировать» (автор/admin), «Удалить рецепт» (автор/admin), «Добавить комментарий» (авторизованный, если ещё не оставлял), «Удалить комментарий» (автор комментария / автор рецепта / admin).
- CRUD-форма рецепта: название, описание, время (мин), сложность (easy/everyday/festive/restaurant/signature), фото (upload — мок), пошаговые инструкции, ингредиенты с количеством, категории (автодополнение, не более одной категории одного типа), флаг публичный/приватный.
- Масштабирование порций: пересчёт ингредиентов на N порций.

### Поиск и фильтрация

- Единое текстовое поле на странице списка рецептов; ищет одновременно по названию, описанию, категориям, ингредиентам (AND-логика по словам).
- Автоподстановка подсказывает категории и ингредиенты.
- Сортировка: по алфавиту / по рейтингу (убывание).
- В сайдбаре переключатель режима: «Все рецепты» / «Избранное».

### Избранное

- Добавление/удаление — только авторизованным.
- Режим «Избранное» в сайдбаре фильтрует список; поиск работает внутри режима.

### Планировщик меню

- Доступен только авторизованным.
- Верхняя панель: горизонтальный список карточек рецептов с поиском и переключателями «Все» / «Избранное» / «Мои».
- Нижняя сетка: 7 дней × 3 приёма пищи (21 слот); в каждый слот можно добавить несколько блюд.
- Карточка блюда в слоте: миниатюра + название + поле порций + кнопка «Удалить».
- Drag-and-drop (HTML5 DnD): перетаскивание из панели в слот.
- Автосохранение при каждом изменении.
- Кнопка «Очистить всё».
- Кнопка «Перейти к списку покупок».

### Список покупок

- Доступен только авторизованным.
- Формируется автоматически из текущего плана меню; обновляется при изменении плана.
- Таблица с группировкой по категориям ингредиентов, сортировка по алфавиту внутри группы.
- Колонки: название, суммарное количество, единица измерения.
- Кнопки: «Скопировать в буфер», «Распечатать» (`window.print()`).
- Состояние empty: план меню пустой.

### Комментарии

- Один комментарий на пользователя на рецепт; редактирование не предусмотрено.
- Максимальная длина — 2000 символов.
- Каждый комментарий: имя автора, дата/время, текст, кнопка «Удалить» (автор комментария / автор рецепта / admin).

### Ингредиенты

- Отдельная страница `/ingredients`: список с группировкой по категориям.
- Фильтры: по названию, по категории; для admin — по автору.
- Форма создания/редактирования ингредиента (title, unit, default_amount, category, is_system).
- Системные ингредиенты редактирует только admin; пользовательские — только создатель.
- Удаление: только если ингредиент не используется в рецептах.

### Категории

- Страница управления категориями (только admin).
- Список категорий с группировкой по типам (meal_role, cooking_method, main_ingredient, cuisine, meal_time, dietary, serving_form).
- Форма создания/редактирования категории (name, description, type).
- Удаление: только если категория не используется в рецептах.

### Дашборд

Виджеты зависят от роли:

**Неавторизованный:** общее количество рецептов (KPI), топ-10 по рейтингу (таблица), гистограмма по «Основному ингредиенту», гистограмма по «Национальной кухне».

**Авторизованный:** + количество своих рецептов (KPI), количество своих комментариев (KPI), топ-10 из избранного по рейтингу, мини-сетка заполненности плана меню.

**Администратор:** + количество пользователей (KPI), количество всех комментариев (KPI), топ-10 пользователей по рейтингам, топ-10 пользователей по комментариям.

Гистограммы — Chart.js.

## Мок-данные

- ≥ 25 рецептов с фото-заглушками (placeholder-сервис или data-URI).
- ≥ 50 ингредиентов (системные, по категориям из clarification.md).
- 3 пользователя: `user1`, `user2`, `admin` (роль admin); логины предзаполнены в форме входа.
- Seed-категории всех типов из `docs/prd/PRD-0001-MVP-clarification.md` (раздел 1).
- Готовый план меню на неделю.
- ≥ 20 комментариев.

## Имитация бэкенда

Модуль `fakeApi` в JS с методами:

```
getRecipes, getRecipe, createRecipe, updateRecipe, deleteRecipe,
login, register, logout,
getMealPlan, updateMealPlan,
getShoppingList,
addComment, deleteComment,
toggleFavorite,
rateRecipe,
getDashboardStats,
getIngredients, createIngredient, updateIngredient, deleteIngredient,
getCategories, createCategory, updateCategory, deleteCategory
```

- Каждый метод возвращает `Promise`, разрешаемый через `setTimeout(…, 150–400 мс)`.
- Данные хранятся в памяти + сериализуются в `localStorage`.

## Результат

- `docs/design/mockup/index.html` открывается двойным кликом в браузере и полностью работоспособен офлайн (с учётом CDN).
- `docs/design/mockup/styles.css` содержит CSS-переменные всех токенов из STYLE_GUIDE.
- Все экраны MVP доступны через навигацию внутри макета.
- Все состояния (empty, loading, error, success) реализованы для каждого экрана.
