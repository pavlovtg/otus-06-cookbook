# MVP-макет «Книга рецептов» в виде HTML

## Роль

- Frontend-прототипировщик: делает кликабельный HTML-макет MVP без реального бэкенда.

## Контекст

- Источник требований: `docs/prd/PRD-0001-MVP.md` (PRD-0001 MVP «Книга рецептов»).
- Backend ещё не реализован — нужны мок-данные и имитация REST-вызовов.
- Макет используется двояко: (1) как инструкция агенту по реализации фронтенда, (2) как основа для последующей стилизации путём заимствования визуального языка стороннего сайта.

## Задача

- Создать единый self-contained файл `docs/design/mockup.html`, покрывающий все пользовательские сценарии MVP.
- Реализовать переходы между экранами внутри одного HTML (hash-роутинг или show/hide секций) — без переходов на отдельные `.html` файлы.
- Все данные — моки в JS; сетевые вызовы — через обёртку `fakeApi.*`, возвращающую `Promise` с `setTimeout` (имитация задержки).

## Требования

### Технические

- Один файл `docs/design/mockup.html`: inline CSS и JS, без npm/сборки, без внешних зависимостей, кроме CDN-ссылок (Chart.js допустим для графика дашборда).
- Язык интерфейса — русский.
- Адаптивная вёрстка (desktop + mobile), минимальные нейтральные стили — готовы к замене.
- В HTML расставить комментарии-маркеры `<!-- style: ... -->` в ключевых местах (header, card, button, form, modal и т.п.) для последующей стилизации.
- Валидация форм на клиенте (required, типы, длина).

### Покрытие функциональности MVP

- Авторизация: формы логина/регистрации, два предзаполненных тестовых пользователя; имитация JWT (токен в `localStorage`); выход.
- Рецепты:
  - список с пагинацией, карточки с фото-заглушкой, именем автора, средним рейтингом, тегами;
  - детальная страница: описание, время, сложность, шаги, ингредиенты, рейтинг (1–5 звёзд), избранное, комментарии;
  - CRUD-форма: название, описание, время, сложность, фото (upload — мок), пошаговые инструкции, ингредиенты с количеством и единицами (г/шт/мл), категории, теги, флаг публичный/приватный;
  - масштабирование порций (пересчёт ингредиентов на N порций).
- Поиск и фильтрация: по названию, по ингредиентам («что приготовить из …»), по тегам/категориям; сортировка по среднему рейтингу.
- Избранное: добавление/удаление, отдельный раздел.
- Планировщик меню: сетка 7 дней × 3 приёма пищи, drag-and-drop рецептов в слоты (HTML5 DnD).
- Список покупок: автогенерация из плана меню с агрегацией ингредиентов и суммированием по единицам.
- Комментарии: добавление авторизованным пользователем, отметка времени и автора; владелец рецепта может удалить любой комментарий.
- Дашборд: визуализация (Chart.js) — например, количество рецептов по категориям, топ-рейтинги, статистика по плану меню.
- Все списки — с пагинацией.

### Мок-данные

- ≥ 25 рецептов с фото-заглушками (placeholder-сервис или data-URI).
- ≥ 50 ингредиентов.
- 2 пользователя (логины предзаполнены в форме входа).
- Готовый план меню на неделю.
- ≥ 20 комментариев.

### Имитация бэка

- Модуль `fakeApi` в JS с методами, отражающими будущие REST-эндпоинты (`getRecipes`, `getRecipe`, `createRecipe`, `updateRecipe`, `deleteRecipe`, `login`, `register`, `getMealPlan`, `updateMealPlan`, `getShoppingList`, `addComment`, `deleteComment`, `toggleFavorite`, `rateRecipe`, `getDashboardStats` и т.п.).
- Каждый метод возвращает `Promise`, разрешаемый через `setTimeout(…, 150–400ms)`.
- Данные хранятся в памяти + сериализуются в `localStorage` для сохранения изменений между перезагрузками.

### Результат

- Файл `docs/design/mockup.html` открывается двойным кликом в браузере и полностью работоспособен офлайн (с учётом CDN).
- Все экраны MVP доступны через навигацию внутри макета.
