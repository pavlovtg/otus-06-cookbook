# Markdown Code Style

## Роль

- Разработчик, создающий стандарт оформления Markdown-файлов для проекта.

## Контекст

- Репозиторий использует Markdown для документации.
- Нужен единый стандарт для людей и AI-агентов.
- Правила основаны на markdownlint (MD009, MD012, MD022, MD032, MD034, MD047).

## Задача

- Создать файл `docs/standards/markdown-code-style.md` со стандартом оформления Markdown.

## Требования

- Структура файла:
  - **Базовые правила** — кодировка UTF-8, отступы 2 пробела, маркер списка `-`, заголовки ATX-стиль.
  - **Правила** — по одному разделу на каждое правило markdownlint: MD009, MD012, MD022, MD032, MD034, MD047.
- Каждое правило: одна строка описания.
- Без ссылок на оригинальную документацию markdownlint.
- Язык: русский.
- Принципы экономии токенов: буллеты вместо прозы, без вводных фраз.
- Сам файл стандарта должен соответствовать описываемым в нём правилам.
