# Инициализация CLine для работы

## Роль

- Ты — разработчик, подготавливающий репозиторий для работы с AI-ассистентом Cline.

## Контекст

- Чистый репозиторий, ничего не настроено.
- В репозитории есть только файлы описывающие общую задачу.
- Проект без настроенных rules, skills, ignore.
- Проект без memory bank и без правил его использования.

## Задачи

- Настроить язык взаимодействия с Cline (русский) и принципы экономии токенов;
- Создать workflow создания промтов;
- Инициализировать memory bank для сохранения контекста между сессиями;
- Ограничить доступ агента к нерелевантным файлам;
- Создать `.gitignore`;
- Создать правило работы с git;

## Требования

Настроить репозиторий для разработки вместе с AI-ассистентом Cline.

1. Язык взаимодействия с агентом и промтинга:
    - Язык взаимодействия c AI-агентом — русский;
    - Cline обязан отвечать **только на русском языке** во всех взаимодействиях, если иное не указано явно пользователем;
    - Cline обязан в ответах использовать **принципы экономии токенов**;
    - Создать правило для Cline `.clinerules/language.md` с языком взаимодействия с агентом и принципами экономии токенов;
2. Настроить workflow `create-prompt`:
    - Создать workflow для Cline `.clinerules/workflows/create-prompt.md` с алгоритмом создания промта.
    - При создании workflow обязан использовать **принципы экономии токенов**;
    - Язык: русский;
    - Зафиксировать в workflow последовательность действий:
        - Спросить у пользователя кодовое название промта (codename);
        - Задать имя файла: `<index>-<codename>.md`, где `<index>` — максимальный существующий индекс + 1 с ведущим нулём, `<codename>` — краткое название латиницей через дефис, запрашивается у пользователя;
        - Создать файл промта в папке `docs/ai/prompts/`;
        - Записать в файл шаблон промта.
    - Зафиксировать в workflow:
        - При создании промта обязан использовать **принципы экономии токенов** (описаны ниже);
        - Язык: русский;
        - Шаблон промта (описан ниже);
        - Структура разделов промта: Роль / Контекст / Задача / Требования, один буллет-плейсхолдер в каждом
3. Настроить правила Cline (`.clineignore`) так, чтобы агент игнорировал нерелевантные части проекта при обходе файловой системы и чтении контекста:
    - Создать файл `.clineignore` с явным перечнем директорий и файлов, которые агент не должен читать без явного запроса пользователя.
    - Включить в список:
        - типовые игнорируемые файлы для .NET (включая результаты тестов и code coverage);
        - типовые игнорируемые файлы для Python (включая результаты тестов, code coverage и cache);
        - бинарные файлы (`*.snk`, `*.png`, `*.svg`, `*.jpg`, `*.ico`);
        - секреты и локальные конфигурации;
        - NuGet (`*.nupkg`,`.nuget/`);
        - Логи `*.log`;
        - файлы типа `.DS_Store`;
    - Оформить файл согласно [.clineignore](https://docs.cline.bot/customization/clineignore).
4. Настроить memory bank:
    - Инициализировать memory bank командой `init memory bank in folder /docs/ai/memory-bank`;
    - Создать `.clinerules/memory-bank.md` — правило читать и обновлять memory bank при каждой задаче;
    - Memory bank на русском языке;
    - При создании/изменении файлов memory bank агент обязан использовать **принципы экономии токенов** (описаны ниже);
5. Настроить git:
    - Создать файл `.gitignore` в корневой папке проекта.
    - В файле `.gitignore` зафиксировать все файлы, которые не нужно отправлять в репозиторий:
        - типовые игнорируемые файлы для .NET (включая результаты тестов и code coverage);
        - типовые игнорируемые файлы для Python (включая результаты тестов, code coverage и cache);
        - бинарные файлы (`*.snk`, `*.png`, `*.svg`, `*.jpg`, `*.ico`);
        - секреты и локальные конфигурации;
        - NuGet (`*.nupkg`,`.nuget/`);
        - Логи `*.log`;
        - файлы типа `.DS_Store`;
6. Настроить правила git commit и git push агентом:
    - Агент **никогда не делает git push самостоятельно** только по команде оператора;
    - Агент **никогда не делает git commit самостоятельно** только по команде оператора;
    - При работе с commit использовать skill - git-commit;
    - Создать правило для Cline `.clinerules/git.md` с этими правилами работы с git';
    - При создании правила обязан использовать **принципы экономии токенов**;
    - Язык: русский;

### Принципы экономии токенов

- Одно предложение вместо абзаца
- Буллеты вместо прозы
- Без вводных фраз ("Конечно!", "Отличный вопрос!")
- Без повторения условия задачи в ответе
- Код без избыточных комментариев

### Шаблон промта

```markdown  
# <Название промта>

## Роль

- <Кто выполняет задачу>.

## Контекст

- <Что известно о ситуации>.

## Задача

- <Что нужно сделать>.

## Требования

- <Ограничения и критерии результата>.
```
