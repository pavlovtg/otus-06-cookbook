# Cookbook Seeder: рефакторинг загрузки начальных данных

## Роль

- Senior .NET-разработчик, знающий проект Cookbook (гексагональная архитектура, EF Core, PostgreSQL).

## Контекст

- Есть два отдельных загрузчика: `IngredientSeeder` и `RecipeSeeder` — каждый работает в своей транзакции EF Core.
- Есть `SeedData.cs` с массивами `Ingredients` и `Recipes` (рецепты без ингредиентов в составе).
- AR-0054 требует upsert-загрузчик; AR-0053 запрещает `HasData`; ADR-0032 — транзакции управляются Application-слоем.
- Текущие загрузчики не управляют порядком загрузки и не гарантируют идемпотентность при частичном сбое.

## Задача

Выполнить три шага последовательно.

### Шаг 1 — Обновить AR-0054

Дополнить `docs/architecture/rules/dotnet/AR-0054-seeding-upsert.md`:

- Один класс-оркестратор управляет порядком и транзакциями всех групп данных.
- Каждая группа данных загружается в отдельной транзакции (ингредиенты → рецепты → связи рецепт-ингредиент).
- Загрузчик идемпотентен: повторный запуск не вызывает ошибок и обновляет изменившиеся данные.

Обновить описание AR-0054 в `ARCHITECTURE.md`.

### Шаг 2 — Реализовать `CookbookSeeder`

- Создать `apps/Cookbook/src/Recipes/Adapters/Postgresql/CookbookSeeder.cs`.
- Удалить `IngredientSeeder.cs` и `RecipeSeeder.cs`.
- Заменить вызовы в `Program.cs` на `await CookbookSeeder.SeedAsync(db)`.
- Структура класса:
  - `SeedIngredientsAsync` — upsert ингредиентов в отдельной транзакции.
  - `SeedRecipesAsync` — upsert рецептов (без ингредиентов) в отдельной транзакции.
  - `SeedRecipeIngredientsAsync` — upsert связей рецепт-ингредиент в отдельной транзакции.
  - `SeedAsync` — оркестратор, вызывает три метода последовательно.
- Каждый метод открывает транзакцию через `db.Database.BeginTransactionAsync`, коммитит в конце или откатывает при исключении:

  ```csharp
  await using var tx = await db.Database.BeginTransactionAsync(cancellationToken);
  // ... upsert ...
  await db.SaveChangesAsync(cancellationToken);
  await tx.CommitAsync(cancellationToken);
  ```

- Upsert: `FindAsync` → `AddAsync` если нет, иначе вызов метода `Update` на сущности.
- Все методы принимают `CancellationToken` последним параметром.

### Шаг 3 — Наполнить `SeedData.cs` ингредиентами рецептов

Добавить в каждый рецепт список `RecipeIngredient` с реалистичными количествами, используя существующие ID ингредиентов из `SeedData.Ingredients`:

- Борщ: свёкла, капуста, картофель, морковь, лук, томатная паста, говядина, сметана, соль, перец, лавровый лист.
- Оливье: картофель, морковь, яйцо, колбаса варёная, майонез, соль, перец.
- Пельмени: мука, яйцо, фарш смешанный, лук, соль, перец.
- Блины: молоко, яйцо, мука, масло сливочное, соль, сахар.
- Котлеты по-киевски: куриное филе, масло сливочное, петрушка, мука, яйцо, панировочные сухари, масло подсолнечное, соль, перец.
- Солянка: говядина, лук, томатная паста, огурец, лимон, соль, перец, лавровый лист.
- Шашлык: свинина, лук, соль, перец, паприка.
- Окрошка: картофель, яйцо, колбаса варёная, огурец, укроп, сметана, соль.
- Плов: рис, говядина (как замена баранины), морковь, лук, масло подсолнечное, соль, куркума, лавровый лист.
- Медовик: мука, яйцо, мёд, масло сливочное, сахар, сметана.

## Требования

- Следовать AR-0038 (один тип — один файл).
- Следовать AR-0054 (upsert, отдельный загрузчик).
- Следовать ADR-0032 (транзакции не в репозитории).
- `CookbookSeeder` — `internal static class`.
- Порядок транзакций: ингредиенты → рецепты → связи (FK-ограничения).
- Все ID ингредиентов в рецептах берутся из `SeedData.Ingredients` (не хардкодить новые GUID).
- Код компилируется без ошибок; стиль — по `docs/standards/csharp-code-style.md`.
