# Активный контекст

## Текущая задача

Оптимизация Docker-сборки контейнеров Cookbook и ApiGateway.

## Что сделано

### Dockerfile оптимизация (Cookbook + ApiGateway)

- Разбит `COPY . .` на два этапа: сначала только `.csproj` → `dotnet restore`, затем исходники → `dotnet publish`
- Добавлен флаг `--no-restore` в `dotnet publish`
- NuGet-пакеты теперь кэшируются в отдельном слое — инвалидируются только при изменении `.csproj`

### .dockerignore (Cookbook)

- Добавлено исключение `seed/photos/` из build-контекста (копируются в runtime-стадии напрямую)

## Ключевые решения

- **Кэширование restore**: слой с `dotnet restore` не инвалидируется при изменении `.cs`-файлов
- **Экономия**: ~60-90 сек на повторных сборках при изменении только кода
- **Структура COPY**: копируются только нужные директории (`Cookbook/src/`, `Shared/src/`), не весь контекст
