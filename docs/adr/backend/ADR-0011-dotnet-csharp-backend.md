# ADR-0011: .NET 10 / C# как платформа и язык backend

- **Статус**: принят
- **Домен**: backend
- **Дата**: 2026-06-07

## Контекст

Проекту требуется единая платформа для реализации всех backend-сервисов: бизнес-логика рецептов, планирования меню, аутентификация, API Gateway.
Команда обладает компетенциями в .NET и C#.

## Рассмотренные варианты

- **.NET 10 / C#** — современный кросс-платформенный стек, нативная поддержка YARP, EF Core, ASP.NET Core, контейнеризация; LTS-перспектива; компетенции команды.
- **Node.js / TypeScript** — широкая экосистема, но дублирует стек frontend и не покрывает компетенции команды по доменному моделированию.
- **Go** — высокая производительность, но слабее по DDD-инструментарию и компетенциям команды.
- **Java / Spring Boot** — зрелая платформа, но команда не имеет профильных компетенций.

## Решение

Все backend-сервисы реализуются на **.NET 10** на языке **C#** (последняя версия языка, поддерживаемая платформой).

## Последствия

- Единый стек упрощает CI, базовые шаблоны проектов, обмен компетенциями.
- Все разработчики backend должны владеть .NET 10 / C#.
- Иные платформы (Node.js, Go, Java и т. п.) для backend-сервисов запрещены без отдельного нового ADR.
- Требуется зафиксировать стандарт оформления C#-кода и структуры .NET-решений.
