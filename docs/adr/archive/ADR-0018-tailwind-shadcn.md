# ADR-0018: Tailwind CSS + shadcn/ui как система стилей и компонентов

- **Статус**: заменён ADR-0031
- **Дата**: 2026-06-07

## Контекст

Frontend требует адаптивной вёрстки (desktop + mobile), готового набора доступных компонентов для ускорения MVP, минимального bundle (initial JS ≤ 200 KB gzip) и AI-friendliness (главный критерий выбора). Альтернативы: CSS Modules + собственные компоненты (больше ручной работы, не вписывается в срок), CSS-in-JS Emotion/styled-components (runtime cost, плохо стыкуется с React Server Components), готовая UI-библиотека MUI/Chakra (больший bundle, меньше гибкости, runtime cost) — отклонены.

## Решение

Стилизация — **Tailwind CSS**. Базовая библиотека компонентов — **shadcn/ui** (компоненты копируются в проект и редактируются под нужды).

## Последствия

- Компоненты shadcn/ui хранятся в репозитории как часть кода (а не как зависимость), их можно менять.
- Tailwind конфигурируется в `tailwind.config.ts`; кастомные токены дизайн-системы — в этом же файле.
- Совместимость с React Server Components — нативная (Tailwind работает на этапе сборки).
- CSS-in-JS-библиотеки запрещены без отдельного нового ADR.
