# ADR-0034: Хранение фотографий рецептов в PostgreSQL

- **Статус**: принят
- **Дата**: 2026-06-14

## Контекст

Рецепты требуют фотографий: загрузка авторизованным пользователем, отображение на карточке и детальной странице, персистентность между перезапусками. Рассматривались три подхода: файловая система с Docker named volume (просто, но shared volume между сервисами нарушает изоляцию), object storage — MinIO (S3-совместимый, но избыточен для учебного MVP), хранение в PostgreSQL в виде бинарных данных (bytea). Альтернативы FS и MinIO отклонены: FS требует shared volume между cookbook-сервисом и nginx, MinIO добавляет дополнительный контейнер и усложняет инициализацию.

## Решение

Фотографии рецептов хранятся в PostgreSQL в виде бинарных данных (`bytea`) в отдельной таблице, принадлежащей cookbook-сервису. Доступ к фото осуществляется через HTTP-endpoint cookbook-сервиса. Docker volume для хранения файлов не используется.

## Последствия

- Нет shared volume между контейнерами — изоляция данных соблюдена.
- Бэкап БД включает фотографии — единая точка восстановления.
- Seed-фото загружаются через seeder в транзакции вместе с рецептами.
- При росте объёма фотографий БД становится узким местом — приемлемо для учебного MVP.
- Каждый запрос фото проходит через backend; HTTP-кэш (`Cache-Control: public`) снижает нагрузку.
