# ADR-0002: Монорепозиторий

- **Статус**: принят
- **Дата**: 2026-06-06

## Контекст

Проект содержит несколько компонентов: документация, спецификации, правила AI-агента, будущий код сервисов. Нужна единая точка управления зависимостями и версионирования. Альтернатива: полирепозиторий (каждый компонент в отдельном репо, независимые релизы, сложнее синхронизировать) — отклонён.

## Решение

Использовать монорепозиторий.

## Последствия

- Все компоненты версионируются вместе.
- Единая точка входа для документации, кода и спецификаций.
- При росте проекта возможен переход к полирепозиторию с созданием нового ADR.
