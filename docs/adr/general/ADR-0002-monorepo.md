# ADR-0002: Монорепозиторий

- **Статус**: принят
- **Домен**: general
- **Дата**: 2026-06-06

## Контекст

Проект содержит несколько компонентов: документация, спецификации, правила AI-агента, будущий код сервисов.
Нужна единая точка управления зависимостями и версионирования.

## Рассмотренные варианты

- **Монорепозиторий** — все компоненты в одном репозитории, единое версионирование, упрощённая навигация.
- **Полирепозиторий** — каждый компонент в отдельном репозитории, независимые релизы, сложнее синхронизировать изменения.

## Решение

Использовать монорепозиторий.

## Последствия

- Все компоненты версионируются вместе.
- Единая точка входа для документации, кода и спецификаций.
- При росте проекта возможен переход к полирепозиторию с созданием нового ADR.
