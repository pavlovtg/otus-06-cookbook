# ADR-0033: Стратегия тестирования

- **Статус**: принят
- **Дата**: 2026-06-12

## Контекст

Монорепо содержит .NET-бэкенд, Next.js-фронтенд и несколько микросервисов за API Gateway. Отсутствует единая политика тестирования: нет требований к уровням пирамиды, покрытию и правилам написания тестов. Альтернативы: только unit-тесты (недостаточная уверенность в интеграциях); только e2e (медленно, хрупко, нет изоляции).

## Решение

Принять семиуровневую пирамиду тестирования:

1. **Unit** — доменная логика, утилиты, схемы в изоляции; внешние зависимости заменяются test-doubles.
2. **Integration (DB)** — взаимодействие с реальной СУБД; используется реальная БД во временном окружении.
3. **Integration (FileSystem)** — взаимодействие с файловой системой без моков; временные директории и файлы.
4. **Integration (classic)** — связки классов внутри сервиса через реальные реализации или test-doubles; только там, где это даёт реальную ценность; проверяется поведение системы, а не факт вызова мока.
5. **Microservice** — полностью работающий сервис с реальной БД, реальной FS и замоканными HTTP-зависимостями других сервисов; тесты на реальном HTTP-стеке; покрывают все публичные API-эндпоинты.
6. **E2E API** — сценарии из `openspec/specs/` против запущенного приложения; покрывают happy path и критичные ошибки.
7. **UI E2E** — важнейшие пользовательские сценарии целиком в браузере.

**Покрытие:** code coverage ≥ 80% по statements на уровне сервиса (unit + integration + microservice тесты суммарно).

**Правило написания тестов:** тесты пишутся сразу после написания кода:

- написали класс → unit-тесты;
- реализовали репозиторий → Integration (DB);
- реализовали работу с FS → Integration (FileSystem);
- сделали API сервиса → Microservice-тесты.

**Запрещено:** тесты, проверяющие только факт вызова мока без проверки поведения системы.

## Последствия

- Единая политика тестирования для всех команд и AI-агентов.
- Конкретные инструменты и фреймворки для каждого уровня определяются в отдельных AR.
- Разделение на 7 уровней увеличивает время прогона, но даёт точную изоляцию и уверенность в каждом слое.
