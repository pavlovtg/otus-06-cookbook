# ADR-0007: Docker Compose как среда развёртывания

- **Статус**: принят
- **Домен**: general
- **Дата**: 2026-06-06

## Контекст

PRD требует, чтобы всё приложение (backend + frontend + БД) запускалось одной командой `docker compose up` в чистом окружении без дополнительной настройки.
Это блокирующий критерий сдачи: невыполнение отправляет проект на доработку.

## Рассмотренные варианты

- **Docker Compose** — декларативное описание многоконтейнерного приложения; стандарт для локального развёртывания; поддерживается Docker Desktop и CI.
- **Kubernetes (Minikube)** — избыточен для учебного проекта; требует дополнительной настройки окружения.
- **Ручной запуск сервисов** — не соответствует требованию одной команды запуска.

## Решение

Выбран Docker Compose как единственная среда развёртывания MVP.

Все сервисы (backend, frontend, PostgreSQL) описаны в `docker-compose.yml`.
Приложение запускается командой `docker compose up` без предварительной настройки.

## Последствия

- Все переменные окружения задаются в `docker-compose.yml` или `.env`-файле, включённом в репозиторий.
- Миграции БД и загрузка seed-данных выполняются автоматически при старте.
- Стандарт реализации: [docs/standards/docker-compose-standard.md](../../standards/docker-compose-standard.md).
- Архитектурное правило: [AR-0001](../rules/general/AR-0001-docker-compose-self-contained.md).
