# ADR-0020: Nginx как edge reverse proxy

- **Статус**: принят
- **Дата**: 2026-06-07

## Контекст

Приложение состоит из Next.js (SPA + BFF) и API Gateway на YARP (ADR-0010), оба запускаются в Docker Compose (ADR-0007). Требуется единая публичная точка входа и кэширование статики (UC: SPA-ассеты, REST API к BFF, Swagger UI). Ограничения: бюджет $0 (только OSS), команда — .NET и Docker, prod на одиночном VPS в РФ без DNS-имени, без compliance, RPO 0 / RTO 5 мин. Целевые показатели: cache hit ratio статики ≥ 90%, p95 для cache hit < 30ms, uptime 99.9%, пик 50 RPS (через год — до 200 RPS). API не кэшируется. TLS — self-signed; ACME не планируется. Альтернативы: Caddy (короткий конфиг, авто-HTTPS, но HTTP-кэш через сторонний плагин и кастомный build), Traefik (динамическая конфигурация через docker labels, но требует доступа к docker.sock и менее зрелый HTTP-кэш), YARP-as-edge (единый .NET-стек, но in-memory кэш без диска и удвоение gateway-слоёв) — отклонены: nginx даёт самую зрелую и предсказуемую модель HTTP-кэша на диске, минимальный overhead и наименьшую сложность при текущем масштабе.

## Решение

В качестве edge reverse proxy используется Nginx (образ `nginx:alpine`), запускаемый отдельным сервисом в `docker-compose.yml` перед Next.js и YARP. Маршрутизация: `/api/*` и `/swagger` проксируются на YARP, остальное — на Next.js. Кэширование выполняется директивой `proxy_cache` на named volume объёмом 1 GB; TTL берётся из заголовков `Cache-Control` upstream-сервисов, при их отсутствии применяется fallback `proxy_cache_valid 200 1h`. Access-логи пишутся в stdout. TLS-терминация — self-signed сертификатом, монтируемым из локальной директории. Reload конфигурации выполняется через рестарт контейнера. Все внешние порты публикуются только edge-сервисом; Next.js и YARP наружу не публикуются.

## Последствия

Положительные: единая публичная точка входа; кэш статики на диске с минимальным overhead (<1ms p95 на целевом RPS); зрелый, широко документированный стек; путь расширения покрыт директивами nginx (gzip/brotli, HSTS, rate-limit) без смены технологии; YARP остаётся внутренним API-gateway согласно ADR-0008/ADR-0010.

Отрицательные: для команды nginx — новый синтаксис конфигурации; hot-reload требует ручного `docker compose restart`; ещё один контейнер в compose и одно дополнительное звено в сетевом пути; кэш-инвалидация — ручная.

Нейтральные: cold-cache после рестарта контейнера допустим в рамках RTO 5 мин; self-signed серт требует исключения в браузере пользователя.
