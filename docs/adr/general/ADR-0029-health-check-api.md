# ADR-0029: Health-check API обязателен для каждого микросервиса

**Статус:** принят  
**Дата:** 2026-06-08

## Контекст

Docker Compose `depends_on: condition: service_healthy` требует, чтобы каждый сервис предоставлял эндпоинт для проверки готовности. Без стандартизированного пути каждый сервис использует произвольный URL (`/healthz`, `/health`, `/ping`), что усложняет конфигурацию и мониторинг.

## Решение

Каждый микросервис обязан реализовать эндпоинт `GET /api/v1/health`. Путь соответствует REST URI-шаблону проекта (`/api/v{major_version}/{resource}`). Эндпоинт возвращает `200 OK` при готовности сервиса.

## Последствия

Конфигурация health-check в Docker Compose унифицирована. Эндпоинт доступен только внутри Docker-сети — наружу не публикуется. Новые сервисы обязаны реализовать `/api/v1/health` до добавления в `docker-compose.yml`.
