# ADR-0004: PostgreSQL как СУБД

- **Статус**: принят
- **Дата**: 2026-06-06

## Контекст

Проект требует реляционной СУБД для хранения рецептов, ингредиентов, пользователей, комментариев и планов меню. PRD допускает PostgreSQL или SQLite, однако требует production-ready поведения: конкурентные запросы, seed-данные при старте, запуск через Docker Compose. Альтернатива: SQLite (встраиваемая БД, ограничена по конкурентной записи, не подходит для многоконтейнерного окружения) — отклонён.

## Решение

PostgreSQL запускается отдельным сервисом в Docker Compose, backend подключается через стандартный DSN. Seed-данные загружаются при старте контейнера.

## Последствия

- Приложение требует запущенного PostgreSQL-сервиса (решается через `depends_on` в Docker Compose).
- Необходимо управление миграциями схемы БД.
- SQLite исключён из рассмотрения для production-окружения.
