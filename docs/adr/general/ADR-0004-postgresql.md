# ADR-0004: PostgreSQL как СУБД

- **Статус**: принят
- **Домен**: general
- **Дата**: 2026-06-06

## Контекст

Проект требует реляционной СУБД для хранения рецептов, ингредиентов, пользователей, комментариев и планов меню.
PRD допускает PostgreSQL или SQLite, однако требует production-ready поведения: конкурентные запросы, seed-данные при старте, запуск через Docker Compose.

## Рассмотренные варианты

- **PostgreSQL** — полноценная реляционная СУБД, поддерживает конкурентные соединения, транзакции, расширения; официальный Docker-образ.
- **SQLite** — встраиваемая БД, не требует отдельного сервиса; ограничена по конкурентной записи, не подходит для многоконтейнерного окружения.

## Решение

Выбран PostgreSQL как основная СУБД проекта.

PostgreSQL запускается отдельным сервисом в Docker Compose, backend подключается через стандартный DSN.
Seed-данные загружаются при старте контейнера.

## Последствия

- Приложение требует запущенного PostgreSQL-сервиса (решается через `depends_on` в Docker Compose).
- Необходимо управление миграциями схемы БД.
- SQLite исключён из рассмотрения для production-окружения.
- Связанный стандарт: [docs/standards/docker-compose-standard.md](../../standards/docker-compose-standard.md) (запуск БД как сервиса).
- Связанное правило: [AR-0001: Docker Compose Self-Contained](../../architecture/rules/general/AR-0001-docker-compose-self-contained.md).
