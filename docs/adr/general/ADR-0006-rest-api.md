# ADR-0006: REST API как протокол взаимодействия

- **Статус**: принят
- **Домен**: general
- **Дата**: 2026-06-06

## Контекст

PRD требует взаимодействия frontend и backend через API с обязательной OpenAPI/Swagger документацией, доступной по живому URL.
Необходим протокол, поддерживающий стандартную документацию и совместимый с экосистемой инструментов проверки (Swagger UI).

## Рассмотренные варианты

- **REST API** — стандарт HTTP-методов и ресурсов; широкая поддержка OpenAPI/Swagger; прост в реализации и тестировании.
- **GraphQL** — гибкие запросы; требует отдельной схемы; Swagger UI не применим напрямую.
- **gRPC** — высокая производительность; требует protobuf; избыточен для учебного проекта с браузерным клиентом.

## Решение

Выбран REST API как протокол взаимодействия frontend–backend.

API документируется через OpenAPI/Swagger; Swagger UI доступен по живому URL без дополнительной настройки.

## Последствия

- Все эндпоинты описываются в OpenAPI-спецификации.
- Swagger UI доступен в запущенном приложении (не статический файл).
- Стандарт реализации: [docs/standards/api-design.md](../../standards/api-design.md).
- Архитектурное правило: [AR-0002: Swagger UI доступен по живому URL](../../architecture/rules/general/AR-0002-swagger-ui-live.md).
