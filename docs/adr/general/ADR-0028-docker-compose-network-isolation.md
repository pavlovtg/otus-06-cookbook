# ADR-0028: Сетевая изоляция сервисов в Docker Compose

**Статус:** принят  
**Дата:** 2026-06-08

## Контекст

В Docker Compose по умолчанию все сервисы помещаются в одну сеть, что позволяет любому сервису обращаться к любому другому. Это нарушает принцип минимальных привилегий: backend-сервисы не должны быть доступны напрямую из frontend-сети, а БД — из frontend-сети вообще.

## Решение

Сервисы разбиваются на сети по принципу безопасности и изоляции:

- `frontend-net`: reverse-proxy, web (BFF), api-gateway.
- `backend-net`: api-gateway, доменные сервисы, БД.

API Gateway принадлежит обеим сетям — он принимает входящие запросы из `frontend-net` и проксирует их в `backend-net`. Только reverse-proxy публикует порты на хост.

## Последствия

Доменные сервисы и БД недоступны из `frontend-net`. Web (BFF) не может напрямую обращаться к доменным сервисам — только через api-gateway. Добавление нового сервиса требует явного указания сети.
