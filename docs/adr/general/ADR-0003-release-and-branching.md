# ADR-0003: Релизная политика и модель ветвления

- **Статус**: принят
- **Домен**: general
- **Дата**: 2026-06-06

## Контекст

Монорепозиторий (ADR-0002: [docs/adr/general/ADR-0002-monorepo.md](ADR-0002-monorepo.md)) требует единой политики выпуска изменений и модели ветвления.

## Рассмотренные варианты

**Релизная политика:**

- Непрерывная доставка (Continuous Delivery) — каждый коммит в `main` потенциально готов к деплою.
- Релизные ветки (Release Branches) — изменения накапливаются в отдельной ветке и выпускаются пакетом.

**Модель ветвления:**

- GitHub Flow — простая модель: `main` + feature-ветки + Pull Request.
- Git Flow — сложная модель с `develop`, `release`, `hotfix`-ветками.
- Trunk-Based Development — все коммиты напрямую в `main`, feature flags для незавершённых фич.

## Решение

Непрерывная доставка + GitHub Flow.

## Последствия

- Простая и понятная модель, подходящая для учебного проекта с небольшой командой.
- Все изменения проходят через Pull Request с обязательным review.
- Сложные модели ветвления (Git Flow) избыточны для данного контекста.
