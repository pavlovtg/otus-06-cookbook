# ADR-0003: Релизная политика и модель ветвления

- **Статус**: принят
- **Дата**: 2026-06-06

## Контекст

Монорепозиторий (ADR-0002) требует единой политики выпуска изменений и модели ветвления. Альтернативы релизной политики: релизные ветки (изменения накапливаются и выпускаются пакетом) — отклонены. Альтернативы модели ветвления: Git Flow (сложная модель с `develop`/`release`/`hotfix`) и Trunk-Based Development (все коммиты в `main`, feature flags) — отклонены.

## Решение

Непрерывная доставка (Continuous Delivery) + GitHub Flow.

## Последствия

- Простая и понятная модель, подходящая для учебного проекта с небольшой командой.
- Все изменения проходят через Pull Request с обязательным review.
- Сложные модели ветвления избыточны для данного контекста.
