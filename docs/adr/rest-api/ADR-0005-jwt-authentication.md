# ADR-0005: JWT-аутентификация

- **Статус**: принят
- **Дата**: 2026-06-06

## Контекст

PRD требует регистрации и авторизации пользователей, разграничения доступа к рецептам (публичные/приватные) и идентификации автора комментариев. Необходим механизм аутентификации, совместимый с REST API и Docker Compose окружением без внешних сервисов. Альтернативы: session-based (требует хранилища сессий или sticky sessions), OAuth2/внешний провайдер (избыточен для учебного проекта без внешних зависимостей) — отклонены.

## Решение

Аутентификация и авторизация через JWT. Access-токен передаётся в заголовке `Authorization: Bearer <token>`. Секретный ключ задаётся через переменную окружения.

## Последствия

- Stateless-архитектура: сервер не хранит состояние сессий.
- Инвалидация токенов до истечения срока требует дополнительного механизма (blacklist) — в MVP не реализуется.
- Два предзаполненных пользователя создаются через seed-данные.
