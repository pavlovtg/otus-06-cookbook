# ADR-0023: Contract-First подход к проектированию API

- **Статус**: принят
- **Дата**: 2026-06-08

## Контекст

Backend-сервисы реализуются на .NET, но потребители API могут быть на любой платформе. Без явного контракта реализация рискует содержать платформо-специфичные артефакты (.NET-дискриминаторы, форматы типов). Альтернативы: code-first (генерация спецификации из кода) — отклонено, т.к. контракт становится производным от реализации и труднее контролируется.

## Решение

API проектируется по принципу contract-first: сначала создаётся OpenAPI-спецификация (YAML, версия ≥ 3.0.0), только после этого пишется реализация. Файлы спецификаций хранятся в `docs/contracts/<bounded-context>/<service-name>.yaml`.

## Последствия

- Контракт API не зависит от платформы реализации.
- Спецификация служит единственным источником истины для генерации клиентов и документации.
- Требует дисциплины: изменения API начинаются с правки YAML-файла, а не кода.
- Swagger UI на API Gateway публикует эти файлы (ADR-0009).
