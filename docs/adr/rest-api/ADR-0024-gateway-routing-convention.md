# ADR-0024: Соглашение о маршрутизации на API Gateway

- **Статус**: принят
- **Дата**: 2026-06-08

## Контекст

YARP выступает единственной точкой входа для API (ADR-0008, ADR-0010). Без явного соглашения о маршрутах каждый сервис может использовать произвольные пути, что усложняет конфигурацию gateway и навигацию по API. Альтернативы: маршрутизация по заголовкам или subdomain — отклонено как избыточное для монорепо с Docker Compose.

## Решение

Маршруты на gateway строятся по шаблону:

```
{gateway}/api/{bounded-context}/... → {service}/api/...
```

Gateway стрипает префикс `/{bounded-context}` перед проксированием. Примеры:

- `/api/recipes/**` → `recipes-service/api/...`
- `/api/auth/**` → `auth-service/api/...`

## Последствия

- Bounded context явно виден в URL — упрощает отладку и документацию.
- Конфигурация YARP унифицирована: один шаблон для всех сервисов.
- Сервисы не знают о своём префиксе на gateway — их маршруты начинаются с `/api/`.
- При добавлении нового сервиса достаточно добавить одно правило в конфигурацию YARP.
