# ADR-0009: Swagger UI публикуется на API Gateway

- **Статус**: принят
- **Дата**: 2026-06-07

## Контекст

ADR-0006 фиксирует REST API как протокол, ADR-0008 вводит API Gateway как единую точку доступа (backend-сервисы становятся внутренними и не публикуются наружу). Внешним потребителям и frontend нужна единая, доступная снаружи точка получения OpenAPI-документации, согласованная с фактической поверхностью API. Альтернативы: Swagger UI на каждом backend-сервисе наружу (конфликтует с ADR-0008), внешний портал документации (рассинхрон с фактическим API, лишний компонент) — отклонены.

## Решение

Swagger UI публикуется на API Gateway и доступен внешним потребителям через ту же точку входа, что и REST API.

- API Gateway — каноничный источник OpenAPI-документации для клиентов системы.
- Backend-сервисы могут предоставлять собственные OpenAPI-документы для внутренних нужд (разработка, тесты), но наружу публикуется только Swagger UI на gateway.
- Способ формирования содержимого Swagger UI на gateway (единый документ или агрегация спецификаций) намеренно не определяется в данном ADR.

## Последствия

- URL Swagger UI у клиента стабилен и не зависит от количества backend-сервисов.
- Требуется отдельное архитектурное решение по способу сборки содержимого Swagger UI (вне скоупа данного ADR).
