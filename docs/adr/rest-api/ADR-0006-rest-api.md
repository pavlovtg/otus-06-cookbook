# ADR-0006: REST API как протокол взаимодействия

- **Статус**: принят
- **Дата**: 2026-06-06

## Контекст

PRD требует взаимодействия frontend и backend через API с обязательной OpenAPI/Swagger документацией, доступной по живому URL. Необходим протокол, поддерживающий стандартную документацию и совместимый с экосистемой инструментов проверки (Swagger UI). Альтернативы: GraphQL (требует отдельной схемы, Swagger UI не применим), gRPC (требует protobuf, избыточен для учебного проекта с браузерным клиентом) — отклонены.

## Решение

API между frontend и backend — REST. API документируется через OpenAPI/Swagger; Swagger UI доступен по живому URL без дополнительной настройки.

## Последствия

- Все эндпоинты описываются в OpenAPI-спецификации.
- Swagger UI доступен в запущенном приложении (не статический файл).
