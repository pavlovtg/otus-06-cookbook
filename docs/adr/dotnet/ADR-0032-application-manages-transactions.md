# ADR-0032: Application-слой управляет транзакциями явно

- **Дата**: 2026-06-12

## Контекст

При реализации операций, изменяющих состояние, необходимо явно управлять границами транзакции. Альтернативы: транзакция внутри репозитория (нельзя объединить несколько операций в одну транзакцию), ambient transaction через `TransactionScope` (плохо работает с async/await) — отклонены.

## Решение

Управление транзакцией — ответственность Application-слоя. Репозитории накапливают изменения, но не фиксируют их самостоятельно. Application-сервис явно фиксирует транзакцию в конце каждого метода, изменяющего состояние.

## Последствия

- Application-сервис контролирует границы транзакции.
- Несколько операций репозитория можно объединить в одну транзакцию.
- Репозитории не фиксируют изменения самостоятельно.
