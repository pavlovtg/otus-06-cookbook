# ADR-0027: WireMock.Net для мокирования HTTP-зависимостей в тестах

**Статус:** принят  
**Дата:** 2026-06-08

## Контекст

.NET-сервисы вызывают внешние HTTP-зависимости (upstream-сервисы, API). В тестах нужно изолировать сервис от реальных зависимостей. Альтернативы: in-process `WebApplication` на случайном порту (сложно управлять lifecycle и верифицировать запросы), подмена `HttpMessageHandler` через DI (зависит от деталей реализации, не проверяет реальный HTTP-стек).

## Решение

Для мокирования HTTP-зависимостей в интеграционных тестах .NET-сервисов использовать `WireMock.Net`. Адрес зависимости переопределяется через `AddInMemoryCollection` в `WithWebHostBuilder` — production-конфигурация не изменяется.

## Последствия

Тесты проверяют реальный HTTP-стек (включая сериализацию, заголовки, трансформацию путей). `WireMock.Net` предоставляет DSL для верификации входящих запросов. Зависимость добавляется только в тестовые проекты.
