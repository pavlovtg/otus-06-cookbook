# ADR-0031: Чистый CSS из Storybook как система стилей фронтенда

**Статус:** принят
**Дата:** 2026-06-11
**Заменяет:** ADR-0018

## Контекст

ADR-0018 зафиксировал Tailwind CSS + shadcn/ui как систему стилей фронтенда. Однако в рамках разработки дизайн-системы (`docs/design/storybook/`) была создана полная система компонентов на чистом CSS с семантическими классами и CSS-переменными. Использование Tailwind совместно с этой системой создаёт два конкурирующих механизма стилизации. Проверка показала, что Tailwind отсутствует в `package.json` и не импортируется в `layout.tsx` — риск удаления минимален. Рассматривались варианты: оставить Tailwind и адаптировать Storybook-компоненты под него; использовать CSS Modules; использовать чистый CSS из Storybook как единственный источник стилей.

## Решение

Система стилей фронтенда основана на чистом CSS из `docs/design/storybook/src/styles.css`. Файл копируется в `apps/web/app/globals.css` и является единственным источником стилей. Компоненты из `docs/design/storybook/src/` переносятся в `apps/web/components/` с адаптацией типов на Zod-схемы. Tailwind CSS и shadcn/ui не используются.

## Последствия

- Единый источник стилей — нет конфликтов между CSS-системами.
- Компоненты из Storybook переносятся 1:1, минимальная адаптация.
- При изменении дизайна требуется ручная синхронизация между Storybook и `apps/web/components/`.
- Tailwind-утилиты недоступны; стилизация только через семантические CSS-классы.
