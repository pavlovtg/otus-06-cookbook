# ADR-0015: Next.js как метафреймворк frontend

- **Статус**: принят
- **Дата**: 2026-06-07

## Контекст

Frontend (ADR-0016) реализуется на React + TypeScript. Требуются SSR/SSG/ISR-режимы для рендеринга страниц рецептов, единая платформа для UI и тонкого серверного BFF-слоя, единый процесс сборки и запуска, хорошая интеграция с экосистемой React. Альтернативы: Remix (меньше распространения, ограниченная экосистема), Vite + ручной SSR-сервер (требует самостоятельной интеграции), Create React App (без SSR, deprecated) — отклонены.

## Решение

Frontend-приложение реализуется как **Next.js**-проект (последняя стабильная мажорная версия) с использованием App Router.

- Один Next.js-процесс выполняет роль и UI-слоя, и серверного BFF-слоя; их разделение фиксируется на уровне модулей (см. ADR-0017).
- TypeScript включён в `strict` режиме (ADR-0016).
- SSR-сервер запускается как отдельный сервис в Docker Compose.

## Последствия

- В монорепо появляется единственное frontend-приложение на Next.js.
- BFF-логика живёт в server actions / route handlers того же приложения, изолированно от client-кода.
- Альтернативный фреймворк/способ запуска фронта требует нового ADR.
