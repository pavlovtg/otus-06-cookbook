# ADR-0016: React + TypeScript как платформа frontend

- **Статус**: принят
- **Дата**: 2026-06-07

## Контекст

Frontend проекта реализуется как самостоятельный сервис с UI поверх REST API (ADR-0006). Требуется зрелая платформа с богатой экосистемой UI-компонентов, типизация для контрактов API и моделей UI, единый язык/рантайм для UI и тонкого серверного BFF-слоя, поддержка SSR через метафреймворк (ADR-0015). Альтернативы: Vue/Nuxt (меньше экосистема UI-китов для текущей задачи), Angular (более тяжёлый и менее гибкий для тонкого BFF), JavaScript без TypeScript (теряем статическую типизацию контрактов API) — отклонены.

## Решение

Frontend строится на **React** (последняя стабильная мажорная версия) с **TypeScript** в `strict` режиме на рантайме **Node.js LTS**.

- Frontend и BFF-слой полностью на TypeScript; JavaScript-файлы в продакшен-коде запрещены, кроме конфигов сборки.
- Версии React и TypeScript обновляются вместе с метафреймворком (ADR-0015).

## Последствия

- AR «Backend — только .NET 10 / C#» не распространяется на frontend и BFF.
- Появляется отдельный TypeScript-стандарт оформления.
- Требуется CI для линтинга и проверки типов в frontend-сборке.
