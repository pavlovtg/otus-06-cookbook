# ADR-0036: @dnd-kit как библиотека drag-and-drop для планировщика меню

- **Статус**: принят
- **Дата**: 2026-06-22

## Контекст

Планировщик меню требует drag-and-drop: пользователь перетаскивает карточки рецептов из горизонтальной панели в слоты сетки 7 колонок × 3 строки (пн–вс × завтрак/обед/ужин). Компонент — Client Component в Next.js App Router. Стек: React + TypeScript strict, чистый CSS (ADR-0031). Рассматривались альтернативы: нативный HTML5 DnD API (нет touch, нет анимаций, кросс-браузерные баги, нет accessibility), react-dnd (большой bundle ~40 KB, touch требует отдельного backend-пакета, активность снизилась), react-beautiful-dnd (заморожена Atlassian с 2023, не поддерживает React 19 и Next.js App Router, ориентирована на сортировку списков, а не на сценарий «панель → сетка»).

## Решение

DnD в планировщике меню реализуется через **`@dnd-kit/core`** и **`@dnd-kit/utilities`**.

- `DndContext` — провайдер на уровне компонента планировщика.
- `useDraggable` — хук на карточке рецепта в панели выбора.
- `useDroppable` — хук на каждом слоте сетки.
- `DragOverlay` — floating-карточка, отображаемая во время перетаскивания.
- `@dnd-kit/utilities` — CSS-трансформы для позиционирования overlay.
- Пакеты `@dnd-kit/sortable` и `@dnd-kit/modifiers` не подключаются: сортировка внутри слота не требуется в MVP.
- Визуальная подсветка слота при наведении — через CSS-класс `is-dragover` из `globals.css`.

## Последствия

- Добавляются две зависимости: `@dnd-kit/core` и `@dnd-kit/utilities` (~15 KB gzip суммарно).
- Touch-события поддерживаются из коробки через Pointer Events API; тестирование touch в MVP не предусмотрено.
- `DragOverlay` рендерится через React Portal — совместимо с Next.js App Router.
- При необходимости сортировки блюд внутри слота в будущем подключается `@dnd-kit/sortable` без замены библиотеки.
- Keyboard navigation и ARIA-атрибуты доступны через `@dnd-kit/core` без дополнительной настройки.
