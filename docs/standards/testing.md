# Стандарт: Тестирование

## Требования

- Минимальное покрытие тестами: ≥ 80%.
- Минимальное количество тестов: не менее 10 unit/integration тестов.
- CI запускает тесты автоматически при каждом push и pull request.

## Типы тестов

- **Unit-тесты** — тестируют отдельные функции и модули в изоляции.
- **Integration-тесты** — тестируют взаимодействие компонентов (например, API-эндпоинты с реальной или тестовой БД).
- **E2E-тесты** — тестируют полные пользовательские сценарии на работающем приложении (браузер + backend + БД); покрывают ключевые flows: регистрация, создание рецепта, поиск, планировщик меню.
