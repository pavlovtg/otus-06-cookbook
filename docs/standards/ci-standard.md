# Стандарт: CI Pipeline

## Платформа

- CI реализован на GitHub Actions.
- Конфигурация хранится в `.github/workflows/`.

## Обязательные шаги

- **Lint** — статический анализ кода; pipeline падает при ошибках линтера.
- **Тесты** — запуск всех unit и integration тестов; pipeline падает при провале любого теста.
- Падающий pipeline блокирует merge pull request.

## Опциональные шаги

- **Сборка Docker-образа** — проверка, что `docker build` завершается успешно.

## Триггеры

- Pipeline запускается при push в любую ветку и при открытии pull request.
