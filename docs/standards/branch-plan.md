# Стандарт ветвления и релизной политики

## Назначение и область применения

Правила работы с ветками и выпуска изменений в монорепозитории `otus-06-cookbook`.

## Правила

### Основная ветка

- `main` — защищённая ветка, прямые коммиты запрещены.
- Каждый коммит в `main` потенциально готов к деплою (Continuous Delivery).

### Feature-ветки

Все изменения вносятся через отдельные ветки и Pull Request.

**Соглашение об именовании:** `<type>/<short-description>`

| Тип | Назначение |
|-----|-----------|
| `feature` | Новая функциональность |
| `bug` | Исправление бага |
| `hotfix` | Быстрый фикс в продакшн |
| `refactor` | Рефакторинг |
| `chore` | Технические изменения (зависимости, конфигурация и т.п.) |

**Примеры:**

- `feature/add-recipe-api`
- `bug/fix-auth-error`
- `hotfix/critical-null-ref`
- `refactor/extract-recipe-service`
- `chore/update-dependencies`

### Pull Request

- Обязательный review перед слиянием.
- Линейная история: squash или rebase (merge commit запрещён).
- Ветка удаляется после слияния.
