# AR-0026: Сетевая изоляция в Docker Compose по принципу безопасности

Источник: ADR-0028

## Правило

Сервисы в `docker-compose.yml` разбиваются на сети по принципу безопасности и изоляции:

- `frontend-net`: reverse-proxy, web (BFF), api-gateway.
- `backend-net`: api-gateway, доменные сервисы, БД.

API Gateway принадлежит обеим сетям. Только reverse-proxy публикует порты на хост.

## Запрещено

- Помещать доменные сервисы или БД в `frontend-net`.
- Публиковать порты сервисов, кроме reverse-proxy.
- Использовать одну общую сеть для всех сервисов.
