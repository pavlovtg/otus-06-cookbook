# AR-0032: Docker Compose — конфигурация через env-переменные

Источник: ADR-0007

## Правило

- Все конфигурируемые параметры docker-compose (имена контейнеров, порты, хосты, credentials) выносятся в `.env`.
- Формат подстановки в `docker-compose.yml` обязателен для всех переменных: `${VAR_NAME:?NO_VAR_NAME}`.
- `container_name` каждого сервиса собирается из `COMPOSE_NAME` как префикс: `${COMPOSE_NAME}-<service>`.
- `ASPNETCORE_URLS` для всех .NET-сервисов задаётся единой переменной через `DEFAULT_HTTP_PORT`.
- `.env` содержит значения по умолчанию для локальной разработки и коммитится в репозиторий.

## Запрещено

- Хардкодить имена контейнеров, порты, хосты, credentials непосредственно в `docker-compose.yml`.
