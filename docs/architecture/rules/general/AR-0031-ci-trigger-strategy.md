# AR-0031: CI — стратегия триггеров

Источник: ADR-0030

## Правило

- **Push в feature-ветку**: запускать lint, test, build только для компонентов, затронутых изменёнными файлами.
- **PR в `main`**: запускать все проверки без фильтров; падение блокирует merge.
- **e2e и ui-тесты**: запускаются только при изменении исполняемого кода (`apps/**`) или конфигурации развёртывания (`docker-compose.yml`, `infrastructure/**`).
- **Markdown lint**: запускается только при изменении `*.md` файлов.

## Запрещено

- Запускать e2e или ui-тесты при изменении только документации, конфигов линтеров или README.
- Пропускать полный прогон на PR в `main`.
