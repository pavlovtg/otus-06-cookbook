# AR-0010: `.editorconfig` — обязательный источник кодстайла

- **Домен**: general
- **Статус**: активно

## Правило

Кодстайл всех файлов в репозитории фиксируется в едином `.editorconfig` в корне репозитория.

- `.editorconfig` обязателен и един для всего монорепо.
- Любая команда/стек, добавляющая код в репозиторий (backend, frontend, scripts, docs), обязана зафиксировать свои правила оформления в этом файле — через секции по glob-паттернам (`[*.cs]`, `[*.ts]`, `[*.md]`, `[*.yml]` и т. п.).
- Альтернативные источники форматирования (локальные настройки IDE, конфликтующие конфиги в подпапках) запрещены, если они расходятся с корневым `.editorconfig`.

## Обоснование

- Единый источник правил исключает «войну форматирования» между разработчиками и IDE.
- EditorConfig поддерживается всеми основными редакторами и инструментами, не требует установки.
- Корневое размещение делает правила обязательными по умолчанию для любого файла в репозитории.

## Нарушение

- Кодстайл стека описан только в README/wiki, а не в `.editorconfig`.
- В подпапке появляется `.editorconfig`, переопределяющий корневые правила без явного обоснования.
- В CI отсутствует проверка соответствия `.editorconfig`.
