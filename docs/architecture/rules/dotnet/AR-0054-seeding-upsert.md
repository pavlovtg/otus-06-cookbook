# AR-0054: Загрузка начальных данных через отдельный класс-загрузчик по принципу Upsert

## Правило

Загрузка начальных данных реализуется отдельным классом-загрузчиком после применения миграций. Загрузчик сравнивает существующие данные с загружаемыми и выполняет upsert. Предпочтительно использовать API репозитория; допустимо выполнять SQL-скрипт для больших объёмов данных.

Один класс-оркестратор управляет порядком и транзакциями всех групп данных. Каждая группа данных загружается в отдельной транзакции (ингредиенты → рецепты → связи рецепт-ингредиент). Загрузчик идемпотентен: повторный запуск не вызывает ошибок и обновляет изменившиеся данные.

## Запрещено

- Совмещать загрузку данных с применением миграций в одном методе.
- Использовать `HasData` (см. AR-0053).
- Иметь несколько независимых классов-загрузчиков без единого оркестратора.
