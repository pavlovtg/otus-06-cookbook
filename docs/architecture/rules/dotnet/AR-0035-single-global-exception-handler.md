# AR-0035: Один глобальный fallback IExceptionHandler для непредвиденных ошибок

## Правило

В каждом .NET-сервисе регистрируется ровно один `IExceptionHandler` — как fallback для непредвиденных исключений. Он возвращает `500 Internal Server Error` с generic-сообщением в формате `application/problem+json`. Никаких других `IExceptionHandler` не добавляется. Доменные исключения перехватываются явно в контроллере (см. AR-0034), а не через глобальный обработчик.

## Запрещено

- Регистрировать более одного `IExceptionHandler`.
- Обрабатывать доменные исключения в глобальном `IExceptionHandler`.
- Возвращать детали внутренних ошибок (stack trace, сообщения исключений) в ответе `500`.
