# AR-0020: Streaming через IAsyncEnumerable вместо буферизации

## Правило

Репозитории и application-сервисы возвращают `IAsyncEnumerable<T>` по умолчанию.

Токен отмены пробрасывается через атрибут `[EnumeratorCancellation]` на параметре метода и `WithCancellation(ct)` внутри `await foreach`.

Контроллеры возвращают `IAsyncEnumerable<T>` напрямую — ASP.NET Core сериализует поток без буферизации.

Допустимо возвращать `IReadOnlyList<T>` только в трёх случаях:

1. Явная задача от оператора.
2. Количество элементов заранее известно и мало (не более 10).
3. Вместе с данными необходимо вернуть `count`, который можно вычислить только после полной материализации коллекции.

## Запрещено

- Буферизовать `IAsyncEnumerable<T>` в `List<T>` / `ToListAsync()` без одного из трёх исключений выше.
- Передавать `CancellationToken` только через `WithCancellation()` без `[EnumeratorCancellation]` — это не пробрасывает токен в итератор.
