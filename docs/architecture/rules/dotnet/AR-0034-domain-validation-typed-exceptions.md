# AR-0034: Доменная валидация через типизированные исключения

Источник: ADR-0012

## Правило

Вся валидация бизнес-правил сосредоточена в доменном слое. Доменные методы выбрасывают типизированное исключение (`RecipeDomainException` или специализированные подклассы) при нарушении инварианта. Контроллер явно перехватывает доменные исключения в `catch`-блоке и формирует `400 Bad Request` с `application/problem+json`. DataAnnotations на DTO для бизнес-валидации не используются.

## Запрещено

- Размещать бизнес-валидацию в Application-слое, контроллере или инфраструктуре.
- Использовать DataAnnotations как замену доменной валидации.
- Пробрасывать доменные исключения через глобальный обработчик как `500 Internal Server Error`.
