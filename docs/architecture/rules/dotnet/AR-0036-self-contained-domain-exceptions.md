# AR-0036: Доменные исключения самодостаточны и не принимают строку сообщения

Источник: ADR-0012

## Правило

Каждое доменное исключение — отдельный класс, наследующий `Exception`. Конструктор без параметров обязателен. Если для понимания ошибки нужен контекст (например, значение, нарушившее инвариант), он передаётся через явные `readonly`-поля, задаваемые в конструкторе. Строковое сообщение через `base(message)` не используется — смысл ошибки выражается именем класса и его полями.

## Запрещено

- Принимать `string message` в конструкторе доменного исключения.
- Использовать базовый класс `RecipeDomainException` как единственное исключение с разными сообщениями.
- Хранить контекст ошибки в строке вместо типизированных полей.
