# AR-0058: Структура unit-тестов frontend

Источник: ADR-0019

## Правило

Для каждого ресурса в `apps/web/lib/` создаются два файла тестов в `apps/web/tests/unit/`:

- `<resource>.schema.test.ts` — тесты Zod-схем: парсинг валидных данных, отклонение невалидных, граничные значения.
- `<resource>.bff.test.ts` — тесты BFF-функций: мокирование `fetch` через `vi.stubGlobal`, проверка метода запроса, query-параметров, возвращаемых данных и ошибок при неуспешном ответе.

## Запрещено

- Объединять тесты схем и BFF в один файл.
- Тестировать BFF без мокирования `fetch`.
- Пропускать тест на ошибку при неуспешном HTTP-ответе.
