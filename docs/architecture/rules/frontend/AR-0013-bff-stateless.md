# AR-0013: BFF stateless

- **Домен**: frontend
- **Статус**: активно

## Назначение

BFF не хранит состояние сессии в собственной памяти или внешнем хранилище. Состояние сессии полностью переносится в подписанном зашифрованном cookie, выдаваемом BFF клиенту.

## Когда применяется

При реализации аутентификации, авторизации и любых пользовательских сессий в BFF.

## Правило

Состояние сессии — только в **signed encrypted cookie**, выдаваемом BFF:

- cookie httpOnly, Secure, SameSite=Lax (минимум);
- подписан и зашифрован секретом, известным только BFF;
- содержит JWT backend или достаточные claims для запросов к backend;
- любой инстанс BFF может обработать запрос без обращения к внешнему хранилищу.

## Разрешено

- Краткосрочный in-memory кэш ответов backend в рамках одного запроса.
- Ротация секрета подписи через переменные окружения.

## Запрещено

- Хранение сессий в Redis, БД или иных внешних хранилищах.
- Серверная in-memory сессия, привязанная к конкретному инстансу BFF.
