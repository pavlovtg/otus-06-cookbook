# AR-0012: BFF не содержит бизнес-логики

- **Домен**: frontend
- **Статус**: активно

## Назначение

BFF-слой отвечает только за нужды UI: агрегацию ответов backend, управление сессией, проксирование. Доменная и бизнес-логика остаётся в backend-сервисах за API Gateway.

## Когда применяется

При проектировании и ревью любого кода в BFF-слое (Server Components, Server Actions, Route Handlers).

## Правило

BFF выполняет только cross-cutting и адаптационные задачи:

- получение JWT при логине и его хранение на серверной стороне;
- выдача и проверка httpOnly session cookie;
- проксирование REST-запросов в YARP Gateway;
- агрегация ответов нескольких backend-эндпоинтов под форму ответа UI;
- CSRF-защита, заголовки безопасности.

Любая бизнес-логика, изменяющая состояние домена, выполняется backend-сервисами.

## Разрешено

- Преобразование DTO backend → ViewModel UI.
- Соединение нескольких backend-ответов в один payload для страницы.
- Валидация входа форм через Zod на стороне BFF перед отправкой в backend.
- Кэширование агрегированных ответов на короткий срок.

## Запрещено

- Реализация доменных инвариантов в BFF.
- Прямой доступ к БД из BFF (PostgreSQL доступна только backend-сервисам).
- Любые бизнес-вычисления, которые должны переживать смену клиента.
