# AR-0017: Database-per-Service — изоляция данных

Источник: ADR-0025

## Правило

Каждый сервис владеет собственной БД и является единственным, кто имеет к ней прямой доступ. Другие сервисы получают данные только через API сервиса-владельца. Именование БД: `<service>` в lowercase snake_case. Внутри БД — одна схема на bounded context (lowercase snake_case); дефолтная схема не используется.

## Запрещено

- Прямой доступ к БД чужого сервиса (кроме сервисов-миграторов данных).
- Использование дефолтной схемы (`public`) для хранения данных приложения.
- Именование БД или схем в стилях, отличных от lowercase snake_case.
