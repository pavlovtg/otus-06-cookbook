# AR-0015: Contract-First — OpenAPI-спецификация до реализации

Источник: ADR-0023

## Правило

Любой новый или изменяемый API MUST быть описан в OpenAPI-спецификации (YAML, версия ≥ 3.0.0) до написания кода реализации. Файл спецификации размещается в `docs/contracts/<bounded-context>/<service-name>.yaml`.

## Запрещено

- Реализовывать эндпоинты без предварительно созданной или обновлённой OpenAPI-спецификации.
- Включать в спецификацию платформо-специфичные артефакты (например, .NET-дискриминаторы `$type` с полными именами типов).
