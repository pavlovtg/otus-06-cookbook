# AR-0005: API Gateway не содержит бизнес-логики

- **Домен**: rest-api
- **Статус**: активно

## Правило

API Gateway отвечает только за cross-cutting задачи: маршрутизацию, авторизацию на входе (валидацию JWT), CORS, rate limiting, наблюдаемость (логи, метрики, трассировки), агрегацию документации. Любая бизнес-обработка — валидация доменных правил, трансформация бизнес-данных, обращение к БД, оркестрация бизнес-сценариев — размещается в backend-сервисах.

## Обоснование

Размещение бизнес-логики на gateway размывает границу ответственности, превращает шлюз в неявный сервис без собственного бэклога и тестов, усложняет миграцию между реализациями gateway и приводит к дублированию правил между gateway и backend.

## Нарушение

- Gateway обращается напрямую к БД или внешним бизнес-системам.
- На gateway реализуется доменная валидация (например, проверка остатков, прав доступа уровня бизнес-сущности).
- На gateway выполняется трансформация бизнес-данных, отличная от технической адаптации формата (например, расчёт скидок, обогащение из БД).
- На gateway появляются собственные доменные модели и репозитории.
