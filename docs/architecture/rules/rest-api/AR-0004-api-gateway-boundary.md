# AR-0004: API Gateway boundary

Источник: ADR-0008, ADR-0010

## Правило

- Frontend и любые внешние потребители обращаются к backend-сервисам ИСКЛЮЧИТЕЛЬНО через API Gateway.
- В среде развёртывания (Docker Compose, любые целевые среды) наружу публикуется ТОЛЬКО порт API Gateway. Backend-сервисы достижимы только во внутренней сети.
- API Gateway выполняет ТОЛЬКО cross-cutting задачи: маршрутизация, авторизация на входе (валидация JWT), CORS, rate limiting, наблюдаемость (логи, метрики, трассировки), агрегация документации.
- Бизнес-логика, обращение к БД, доменные модели и репозитории на gateway не размещаются.

## Запрещено

- URL backend-сервиса (имя контейнера, прямой порт) в frontend-коде или у внешнего потребителя в обход gateway.
- `ports:` у backend-сервиса в `docker-compose.yml` с пробросом на хост.
- Любой запрос «снаружи» успешно достигает backend, минуя gateway.
- Прямой доступ gateway к БД или внешним бизнес-системам.
- Доменная валидация на gateway (проверка остатков, прав уровня бизнес-сущности).
- Трансформация бизнес-данных на gateway, отличная от технической адаптации формата (расчёт скидок, обогащение из БД).
- Собственные доменные модели и репозитории на gateway.
