# AR-0013: JWT-валидация обязательна в recipes-сервисе

Источник: ADR-0005, ADR-0035

## Правило

`recipes`-сервис обязан валидировать JWT (подпись, `iss`, `aud`, `exp`) на всех защищённых маршрутах. YARP не выполняет JWT-валидацию и является чистым прокси.

## Запрещено

- Доверие заголовкам `Authorization`/claims без проверки подписи.
- Отключение `JwtBearer`-middleware на защищённых маршрутах.
- Передача user-claims downstream-у в отдельных HTTP-заголовках вместо самого JWT.
