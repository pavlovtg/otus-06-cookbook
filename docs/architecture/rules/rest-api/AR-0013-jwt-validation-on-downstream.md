# AR-0013: JWT-валидация обязательна на каждом downstream-сервисе

Источник: ADR-0005, ADR-0021

## Правило

Каждый backend-сервис за API Gateway обязан повторно валидировать JWT (подпись, `iss`, `aud`, `exp`) для всех защищённых маршрутов. Валидация на YARP не освобождает downstream от собственной проверки.

## Запрещено

- Доверие заголовкам `Authorization`/claims без проверки подписи.
- Отключение `JwtBearer`-middleware (или эквивалента) на защищённых маршрутах.
- Передача user-claims downstream-у в отдельных HTTP-заголовках вместо самого JWT.
