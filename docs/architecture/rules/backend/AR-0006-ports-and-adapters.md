# AR-0006: Внешние интеграции — через порты и адаптеры

Источник: ADR-0013

## Правило

Любое взаимодействие backend-сервиса с внешним миром (БД, HTTP-клиенты, очереди, файловая система, внешние API, время, генерация идентификаторов и т. п.) выполняется только через **порты**, объявленные в слое `Application`, и **адаптеры**, реализующие эти порты.

- Порт = интерфейс (или абстракция) в `Application`.
- Адаптер = реализация порта в `Adapters/<technology>` (например, `Adapters/Postgresql`, `Adapters/Web`).
- Слои `Domain` и `Application` не зависят от конкретных технологий (EF Core, ASP.NET, SDK внешних сервисов).

## Запрещено

- В `Domain` или `Application` импортируется тип из EF Core, ASP.NET, Npgsql, HttpClient и т. п.
- Доменный код напрямую обращается к БД, файловой системе, сети.
- Адаптер вызывает другой адаптер напрямую, минуя `Application`.
