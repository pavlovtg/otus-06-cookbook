# AR-0008: Видимость типов — `internal` по умолчанию

## Правило

В backend-сборках все типы по умолчанию объявляются с модификатором `internal`. Модификатор `public` допустим **только** для типов, являющихся **публичным API сборки** — то есть типов, на которые ссылаются из других сборок.

- Сервисы (executable, `Program.cs`) и утилиты (CLI-инструменты) не имеют публичного API — все их типы должны быть `internal`.
- Библиотеки (`*.csproj` без точки входа, переиспользуемые между сборками) имеют публичное API только в виде типов, действительно используемых снаружи. Остальные типы — `internal`.
- Для доступа из тестовых сборок к `internal`-типам использовать `InternalsVisibleTo`.

## Запрещено

- `public` у типа в сборке-сервисе или сборке-утилите.
- `public` у внутренней реализации в библиотеке, не использующейся из других сборок.
- Тестовый проект напрямую ссылается на `public`-проброс типов, сделанный «чтобы тесты увидели».
