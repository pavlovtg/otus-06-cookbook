# План разработки MVP — Книга рецептов

План реализации функциональных и нефункциональных требований из `PRD-0001-MVP.md`. Этапы расположены в порядке реализации: зависимости ссылаются только на предыдущие номера. Каждый этап — законченный блок работы для кросс-функциональной команды.

## Легенда

- `[бизнес]` — фича из функциональных требований PRD.
- `[тех]` — нефункциональные требования: инфраструктура, ADR, макеты, документация, тесты, CI.

## 1. [тех] Инициализация монорепо и базовых стандартов

- [x] выполнено

Описание: создан монорепозиторий, зафиксированы стандарты структуры, кода и Markdown, заведена документация и шаблоны.

Зависимости: нет.

## 2. [тех] Зафиксировать ключевые ADR и AR

- [x] выполнено

Описание: приняты решения по стеку (.NET 10, Next.js, PostgreSQL, JWT), архитектуре (DDD, гексагон, API Gateway, edge reverse proxy, database-per-service) и оформлены соответствующие AR и стандарты.

Зависимости: 1.

## 3. [тех] Docker Compose как среда запуска

- [x] выполнено

Описание: backend, frontend, БД и reverse proxy поднимаются одной командой `docker compose up`; конфигурация вынесена в `.env`, сети изолированы.

Зависимости: 2.

## 4. [тех] Health-check API на всех сервисах

- [x] выполнено

Описание: каждый сервис отдаёт health-check; используется в `depends_on: service_healthy` Docker Compose.

Зависимости: 3.

## 5. [тех] CI pipeline

- [x] выполнено

Описание: GitHub Actions для push и PR; lint и тесты по стекам; полный прогон при PR в main.

Зависимости: 1.

## 6. [тех] Contract-First OpenAPI и Swagger UI

- [x] выполнено

Описание: контракты сервисов описаны в OpenAPI YAML до реализации; Swagger UI публикуется на API Gateway как живая веб-страница.

Зависимости: 2.

## 7. [тех] Макеты ключевых экранов рецептов

- [x] не выполнено

Описание: wireframes списка рецептов, карточки рецепта, форм создания/редактирования; согласована навигация и адаптивные брейкпойнты.

Зависимости: нет.

## 8. [бизнес] CRUD рецептов

- [x] не выполнено

Описание: создание, просмотр, редактирование, удаление рецепта с полями «название, описание, время приготовления, сложность, пошаговые инструкции»; валидация на клиенте и сервере.

Зависимости: 7.

## 9. [бизнес] Ингредиенты рецепта

- [ ] не выполнено

Описание: добавление ингредиентов к рецепту с количеством и единицами измерения (граммы, штуки, мл); отображение в карточке.

Зависимости: 8.

## 10. [тех] ADR «хранилище загружаемых файлов»

- [ ] не выполнено

Описание: зафиксировать выбор подхода к хранению пользовательских изображений (файловое хранилище, БД, S3-совместимое и т.п.) с обоснованием и последствиями.

Зависимости: нет.

## 11. [бизнес] Загрузка фото рецепта

- [ ] не выполнено

Описание: пользователь прикладывает фото к рецепту; фото отображается в списке и карточке; ограничения по размеру и формату.

Зависимости: 8, 10.

## 12. [бизнес] Категории и теги рецептов

- [ ] не выполнено

Описание: справочник категорий и тегов (завтрак, десерт, веган, быстрое и т.д.); рецепту назначаются теги; отображение тегов в карточке.

Зависимости: 8.

## 13. [бизнес] Пагинация списков рецептов

- [ ] не выполнено

Описание: список рецептов разбивается на страницы; есть навигация по страницам и индикация общего количества.

Зависимости: 8.

## 14. [бизнес] Поиск и фильтрация рецептов

- [ ] не выполнено

Описание: поиск по названию, тегам и набору ингредиентов («что приготовить из курицы и риса»); комбинирование фильтров.

Зависимости: 9, 12, 13.

## 15. [бизнес] Масштабирование порций

- [ ] не выполнено

Описание: пользователь задаёт количество порций; количества ингредиентов пересчитываются пропорционально.

Зависимости: 9.

## 16. [тех] Auth-service как issuer JWT

- [ ] не выполнено

Описание: реализовать выделенный сервис аутентификации, выпускающий JWT для пользователей и сервис-к-сервис вызовов; интеграция с API Gateway.

Зависимости: нет.

## 17. [бизнес] Регистрация и авторизация пользователей

- [ ] не выполнено

Описание: формы регистрации и входа; JWT-сессия; 2 предзаполненных тестовых пользователя; защищённые маршруты.

Зависимости: 16.

## 18. [бизнес] Публичные и приватные рецепты, отображение автора

- [ ] не выполнено

Описание: при создании рецепта выбирается видимость (публичный/приватный); приватный виден только автору; имя автора показывается на карточке.

Зависимости: 8, 17.

## 19. [бизнес] Избранное

- [ ] не выполнено

Описание: авторизованный пользователь добавляет рецепты в закладки; есть отдельный список «Избранное».

Зависимости: 8, 17.

## 20. [бизнес] Рейтинг рецептов

- [ ] не выполнено

Описание: пользователь выставляет рецепту оценку 1–5 звёзд; в списке доступна сортировка по среднему рейтингу.

Зависимости: 8, 17.

## 21. [бизнес] Комментарии к рецептам

- [ ] не выполнено

Описание: авторизованный пользователь оставляет комментарии с автором и отметкой времени; владелец рецепта удаляет любой комментарий.

Зависимости: 8, 17.

## 22. [тех] ADR «drag-and-drop библиотека»

- [ ] не выполнено

Описание: зафиксировать выбор DnD-решения для планировщика меню с обоснованием и ограничениями.

Зависимости: нет.

## 23. [тех] Макет планировщика меню

- [ ] не выполнено

Описание: wireframe недельной сетки (пн–вс × завтрак/обед/ужин), взаимодействие drag-and-drop, поведение на мобильных.

Зависимости: 22.

## 24. [бизнес] Планировщик меню на неделю

- [ ] не выполнено

Описание: пользователь перетаскивает рецепты в слоты недельной сетки; план сохраняется за пользователем.

Зависимости: 8, 17, 22, 23.

## 25. [бизнес] Автогенерация списка покупок

- [ ] не выполнено

Описание: на основе плана меню формируется список покупок с агрегацией и суммированием ингредиентов по единицам измерения.

Зависимости: 9, 24.

## 26. [бизнес] Дашборд со статистикой

- [ ] не выполнено

Описание: визуализация данных пользователя (количество рецептов, распределение по категориям, активность, средний рейтинг и т.п.) графиками и метриками.

Зависимости: 8, 17, 20, 24.

## 27. [бизнес] Адаптивная вёрстка

- [ ] не выполнено

Описание: все экраны корректно отображаются и работают на desktop и mobile; проверены ключевые сценарии на узких брейкпойнтах.

Зависимости: 8, 11, 13, 14, 17, 18, 19, 20, 21, 24, 25, 26.

## 28. [тех] Seed-данные

- [ ] не выполнено

Описание: при запуске приложение содержит 25+ рецептов с фото-заглушками, 50+ ингредиентов, готовый план меню на неделю, 2 пользователей, 20+ комментариев.

Зависимости: 9, 11, 12, 17, 21, 24.

## 29. [тех] Покрытие тестами ≥ 80%

- [ ] не выполнено

Описание: не менее 10 unit/integration тестов суммарно по бизнес-фичам; целевое покрытие 80%; интеграция в CI.

Зависимости: 8, 14, 15, 17, 18, 19, 20, 21, 24, 25.

## 30. [тех] Финализация `README.md`

- [ ] не выполнено

Описание: пошаговая инструкция запуска `docker compose up`, описание основных URL (приложение, Swagger UI), учётные данные seed-пользователей.

Зависимости: 3.

## 31. [тех] `REPORT.md` — история проекта

- [ ] не выполнено

Описание: ведётся по ходу работы; описывает ключевые проблемы и решения, использованные AI-инструменты, минимум один показательный пример промпта с разбором.

Зависимости: нет.

## 32. [тех] Smoke-проверка запуска в чистом окружении

- [ ] не выполнено

Описание: на чистой машине `git clone && docker compose up` поднимает приложение менее чем за 2 минуты; seed-данные доступны, Swagger UI открыт, основные сценарии работают.

Зависимости: 27, 28, 29, 30.
