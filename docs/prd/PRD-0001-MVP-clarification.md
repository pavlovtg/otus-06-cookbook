# PRD-0001-MVP: Уточнения требований

Документ фиксирует уточнённые требования к MVP «Книга рецептов» для использования при создании макета и Storybook.
Каждый раздел — результат диалога; финальная формулировка однозначна и не требует дополнительных уточнений.

---

## 1. Категории и теги рецептов

### Модель данных

- Категория — сущность с полями: `id`, `name`, `description`, `type`.
- Тип категории (`type`) — фиксированный enum, не редактируется. Допустимые типы:
  - `meal_role` — роль в приёме пищи
  - `cooking_method` — способ приготовления
  - `main_ingredient` — основной ингредиент
  - `cuisine` — национальная кухня
  - `meal_time` — время употребления
  - `dietary` — диетические особенности
  - `serving_form` — форма подачи
- Рецепту можно присвоить любое количество категорий (включая 0), но **не более одной категории одного типа**.
- Категории — это и есть теги; отдельной сущности «тег» нет.

### Управление категориями

- Создавать, редактировать и добавлять категории может только **администратор**.
- Удалить категорию можно только если она **не используется ни в одном рецепте**; иначе — только редактирование.
- Типы категорий редактировать нельзя.

### Seed-данные (категории из коробки)

**Роль в приёме пищи:** Закуски холодные, Закуски горячие, Первые блюда, Вторые блюда, Гарниры, Салаты, Десерты, Напитки, Соусы и заправки.

**Способ приготовления:** Варёные, Жареные, Запечённые, Тушёные, На пару, Гриль/барбекю, Копчёные, Маринованные, Сырые.

**Основной ингредиент:** Мясные, Рыбные, Из морепродуктов, Птичьи, Овощные, Грибные, Яичные, Молочные, Зерновые и крупяные, Бобовые, Фруктовые.

**Национальная кухня:** Итальянская, Французская, Японская, Китайская, Индийская, Мексиканская (и другие — добавляются администратором).

**Время употребления:** Завтраки, Бранчи, Обеды, Ужины, Перекусы, Праздничные блюда.

**Диетические особенности:** Вегетарианские, Веганские, Безглютеновые, Низкоуглеводные, Кето, Диетические, Постные.

**Форма подачи:** Супы, Рагу, Запеканки, Пироги, Пиццы, Сэндвичи, Паста, Роллы, Каши, Блины, Торты и пирожные.

### UI: выбор категорий при создании/редактировании рецепта

- Единый инпут с поиском по названию категории (автодополнение).
- При выборе категории она добавляется как тег под инпутом.
- Опционально: дерево выбора, сгруппированное по типам (раскрывающиеся группы).
- Если пользователь выбирает категорию того же типа, что уже назначена — новая **заменяет** существующую (без ошибки).

### UI: поиск/фильтрация рецептов по категориям

- На странице списка рецептов — текстовое поле поиска (общий поиск по названию, ингредиентам, категориям).
- Автоподстановка подсказывает совпадающие категории из справочника.
- Отдельных фильтров-чекбоксов по типам категорий нет.

## 2. Поиск рецептов

### Расположение

- Поиск находится на странице списка рецептов — единое текстовое поле, которое фильтрует отображаемый список в реальном времени (или по нажатию Enter).
- Отдельной страницы результатов поиска нет.

### Поля поиска

Одна строка ищет одновременно по:

- `title` — название рецепта
- `description` — описание рецепта
- `categories.name` — названия категорий, присвоенных рецепту
- `ingredients.name` — названия ингредиентов рецепта

Отдельных полей для выбора категории или ингредиентов нет.

### Логика совпадения

- Слова в строке поиска интерпретируются как **AND**: каждое слово должно найтись хотя бы в одном из перечисленных полей.
- Если введено несколько слов, соответствующих ингредиентам — рецепт должен содержать **все** эти ингредиенты.
- Пример: запрос «курица рис» вернёт только рецепты, у которых есть и «курица», и «рис» (в любом из полей).

### Автоподстановка

- Желательна: при вводе показываются подсказки из названий категорий и ингредиентов.

## 3. Рецепты: карточка и детальная страница

### Модель данных рецепта

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `id` | uuid | да | |
| `title` | string | да | Название |
| `description` | string | нет | Описание |
| `difficulty` | enum | да | Сложность (см. ниже) |
| `cooking_time` | int (мин) | да | Время приготовления в минутах |
| `photos` | string[] | нет | Галерея URL; если пусто — показывается заглушка |
| `instructions` | string | нет | Пошаговые инструкции, одно текстовое поле (в перспективе Markdown) |
| `servings` | int | да | Количество порций (базовое) |
| `is_public` | bool | да | Публичный / приватный рецепт |
| `author_id` | uuid | да | Автор рецепта |
| `categories` | Category[] | нет | Присвоенные категории (0 или более) |
| `ingredients` | RecipeIngredient[] | нет | Ингредиенты с количеством и единицами |

**Степени сложности (`difficulty`):**

- `easy` — Простые
- `everyday` — Повседневные
- `festive` — Праздничные
- `restaurant` — Ресторанные
- `signature` — Авторские

### Публичность рецепта

- Признак `is_public` задаётся при создании и редактировании рецепта.
- **Публичный** рецепт: доступен для поиска и просмотра всем пользователям, включая неавторизованных.
- **Приватный** рецепт: доступен для поиска и просмотра только автору и администратору.
- На карточке приватного рецепта отображается визуальный тег «Приватный» (аналогично тегу категории); у публичного — ничего не отображается.
- На детальной странице признак публичности также отображается.

### Карточка рецепта (в списке)

Отображает:

- Фото (первое из галереи) или заглушка
- `title`
- Категории (теги)
- Время приготовления
- Сложность
- Имя автора
- Общий рейтинг (средняя оценка)
- Мой рейтинг (если текущий пользователь уже оценил)
- Кнопка/иконка «Избранное»
- Тег «Приватный» (если `is_public = false`)

При клике на карточку — переход на детальную страницу рецепта (без промежуточного попапа).

### Детальная страница рецепта

**Контент:**

- Галерея фото (или заглушка)
- `title`, `difficulty`, `cooking_time`, автор, категории, признак публичности
- `description`
- Ингредиенты с количеством и единицами (с возможностью пересчёта порций)
- `instructions` (пошаговые инструкции)
- Рейтинг и комментарии

**Органы управления:**

- «Назад» — возврат на страницу списка рецептов с сохранением позиции прокрутки и поискового запроса
- «В избранное» — добавить/убрать
- «Задать рейтинг» — выставить оценку 1–5 звёзд
- «Пересчитать порции» — изменить количество порций, ингредиенты пересчитываются
- «Редактировать» — только для автора или администратора
- «Удалить рецепт» — только для автора или администратора
- «Добавить комментарий» — только для авторизованных пользователей
- «Удалить комментарий» — кнопка напротив каждого комментария, видна только автору рецепта или администратору

## 4. Избранное

### Поведение

- Добавить/убрать рецепт из избранного может только **авторизованный** пользователь.
- Неавторизованный пользователь кнопку «В избранное» **не видит** (ни на карточке, ни на детальной странице).
- Избранное хранится на сервере, привязано к аккаунту пользователя.

### UI на странице списка рецептов

- В **сайдбаре** есть переключатель режима просмотра: «Все рецепты» / «Избранное».
- При выборе «Избранное» список показывает только рецепты, добавленные текущим пользователем в избранное.
- Поиск (текстовое поле) работает в обоих режимах — при активном режиме «Избранное» поиск ищет только среди избранных рецептов.

### Использование в планировщике меню

- На странице планировщика меню избранное используется как один из способов выбора блюда для слота (подробнее — в разделе «Планировщик меню»).

## 5. Рейтинг рецептов

### Модель

- Рейтинг — оценка от 1 до 5 звёзд, выставленная конкретным пользователем конкретному рецепту.
- Значение 0 означает «не задан» (рейтинг не существует).
- Один пользователь — одна оценка на рецепт; повторная простановка **заменяет** предыдущую.
- Общий рейтинг рецепта = среднее арифметическое всех оценок пользователей (округление — на усмотрение реализации).

### Кто видит рейтинг

- Общий рейтинг виден **всем**, включая неавторизованных пользователей (на карточке и детальной странице).

### Кто может ставить рейтинг

- Любой **авторизованный** пользователь, включая автора рецепта.
- Неавторизованный пользователь не может ставить оценку (элемент простановки скрыт или заблокирован).

### UI

- Простановка звёзд — отдельное действие, не связанное с редактированием рецепта (inline на детальной странице).
- На карточке и детальной странице отображается:
  - Общий рейтинг (среднее, визуально — звёзды или число).
  - Рядом — оценка текущего пользователя (если он авторизован и уже оценил рецепт).

### Сортировка списка рецептов

Список рецептов поддерживает сортировку по:

- `title` — по алфавиту (A→Я).
- `rating` — по среднему рейтингу (убывание).

Переключатель сортировки расположен на странице списка рецептов (рядом с поиском или в сайдбаре).

## 6. Комментарии к рецептам

### Модель

- Комментарий содержит: `id`, `recipe_id`, `author_id`, `text`, `created_at`.
- Максимальная длина текста — **2000 символов**.
- Пользователь может оставить **только один** комментарий к каждому рецепту.
- Редактирование комментария **не предусмотрено** — только удаление.

### Права доступа

| Действие | Кто может |
|----------|-----------|
| Просматривать комментарии | Все (включая неавторизованных) |
| Оставить комментарий | Авторизованный пользователь (если ещё не оставлял к данному рецепту) |
| Удалить свой комментарий | Автор комментария |
| Удалить любой комментарий | Автор рецепта или администратор |

### UI

- Комментарии отображаются на детальной странице рецепта.
- Каждый комментарий показывает: имя автора, дату/время, текст.
- Кнопка «Удалить» напротив комментария видна: автору этого комментария, автору рецепта, администратору.
- Форма добавления комментария видна только авторизованным пользователям, у которых ещё нет комментария к данному рецепту.

## 7. Авторизация и роли

### Роли

В системе три уровня доступа:

| Роль | Описание |
|------|----------|
| Неавторизованный | Гость, не вошедший в систему |
| Авторизованный пользователь | Зарегистрированный и вошедший пользователь |
| Администратор | Авторизованный пользователь с ролью `admin` |

Роль администратора назначается через seed-данные; интерфейс управления ролями не предусмотрен.

### Модель пользователя

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | uuid | |
| `email` | string | Логин; уникальный |
| `display_name` | string | Имя для отображения в карточках, комментариях, детальной странице |
| `password_hash` | string | Хэш пароля |
| `role` | enum (`user` / `admin`) | Роль пользователя |

### Валидация при регистрации

- `display_name`: от 3 до 200 символов.
- `email`: валидный формат email; верификация через письмо не требуется.
- `password`: длиннее 8 символов; дополнительных требований к сложности нет.

### Доступ по ролям (сводная таблица)

| Функция | Неавторизованный | Авторизованный | Администратор |
|---------|-----------------|----------------|---------------|
| Просмотр списка публичных рецептов | ✅ | ✅ | ✅ |
| Поиск по публичным рецептам | ✅ | ✅ | ✅ |
| Просмотр детальной страницы публичного рецепта | ✅ | ✅ | ✅ |
| Просмотр приватных рецептов | ❌ | Только свои | ✅ |
| Регистрация / вход | ✅ | — | — |
| Создание рецепта | ❌ | ✅ | ✅ |
| Редактирование/удаление рецепта | ❌ | Только свои | ✅ (любые) |
| Избранное | ❌ | ✅ | ✅ |
| Рейтинг (простановка) | ❌ | ✅ | ✅ |
| Комментарии (добавление) | ❌ | ✅ | ✅ |
| Удаление комментария | ❌ | Только свой | ✅ (любой) |
| Планировщик меню | ❌ | ✅ | ✅ |
| Список покупок | ❌ | ✅ | ✅ |
| Управление категориями | ❌ | ❌ | ✅ |
| Управление системными ингредиентами | ❌ | ❌ | ✅ |
| Управление пользовательскими ингредиентами | ❌ | ✅ (только свои, если не используются) | ✅ |

### Сессия и токены

- Аутентификация через JWT.
- При получении ответа `401` от API — фронт предлагает пользователю авторизоваться (модальное окно входа или редирект на страницу логина).
- Логаут: сессия удаляется, интерфейс перестраивается в режим неавторизованного пользователя.

### Ингредиенты: системные и пользовательские

- **Системные** ингредиенты: создаёт и редактирует только администратор; удалить нельзя если используются в рецептах.
- **Пользовательские** ингредиенты:
  - Создаёт любой авторизованный пользователь.
  - Пользователь видит при поиске/добавлении в рецепт: **системные + только свои** пользовательские.
  - Использовать в рецепте можно только системные и **свои** пользовательские ингредиенты.
  - Удалить может только **создатель** ингредиента — и только если он **не используется ни в одном рецепте**.
- Детали UI управления ингредиентами — в отдельном разделе.

### UI

- Элементы управления, недоступные текущей роли, **не отображаются** (скрываются, не блокируются).
- API возвращает `403` при попытке выполнить недоступную операцию, `401` при отсутствии/истечении токена.

## 8. Ингредиенты

### Модель данных

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `id` | uuid | да | |
| `title` | string (мин. 2 символа) | да | Название ингредиента |
| `unit` | string (1–20 символов) | да | Единица измерения; текстовое поле, пример в тултипе (напр. «г», «мл», «шт») |
| `default_amount` | decimal | да | Дефолтное количество (по умолчанию: 100) |
| `category` | enum | да | Категория ингредиента (см. ниже) |
| `is_system` | bool | да | Системный (`true`) или пользовательский (`false`) |
| `author_id` | uuid / null | нет | Автор; заполняется автоматически; системные могут быть без автора |

**Единицы измерения** берутся из ингредиента и не могут быть переопределены при добавлении в рецепт.

**Дефолтные значения:** `unit` — `г`, `default_amount` — `100`.

### Категории ингредиентов

Фиксированный список, не редактируется. Используется для навигации и форматирования вывода:

- Овощи
- Фрукты и ягоды
- Мясо и птица
- Рыба и морепродукты
- Молочные продукты и яйца
- Крупы и зерновые
- Бобовые
- Орехи и семена
- Масла и жиры
- Специи и приправы
- Соусы и пасты
- Выпечка и сладости
- Другое

### Количество ингредиента в рецепте

- Всегда положительное число > 0.
- Может быть дробным, не более 3 знаков после запятой.
- Максимальное значение: `100 000`.
- Количество указывается на базовое число порций рецепта (`servings`).

### Права доступа

| Действие | Неавторизованный | Авторизованный | Администратор |
|----------|-----------------|----------------|---------------|
| Просмотр каталога | ✅ | ✅ | ✅ |
| Использование в рецепте | ❌ | Системные + свои пользовательские | ✅ |
| Создание пользовательского ингредиента | ❌ | ✅ | ✅ |
| Редактирование системного ингредиента | ❌ | ❌ | ✅ |
| Редактирование пользовательского ингредиента | ❌ | Только свои | ✅ (любые) |
| Удаление ингредиента | ❌ | Только свои, если не используется | ✅, если не используется |

### Удаление ингредиента

- Нельзя удалить ингредиент, используемый хотя бы в одном рецепте.
- При попытке удаления — ошибка с перечнем рецептов (`title`), где он используется: топ-10 + общий счётчик использований.

### UI: страница каталога ингредиентов

- Отдельная страница в навигации (`/ingredients`).
- Отображает список ингредиентов с группировкой по категориям.
- Фильтры на странице:
  - По названию — для всех.
  - По категории — для всех.
  - По автору — только для администратора (свободный выбор любого пользователя). Для авторизованного пользователя фильтр по автору **зафиксирован на его собственном id** (UI показывает автора, но не дает выбирать); API принимает только `author_id` текущего пользователя, иначе возвращает `403`.
- Форма создания/редактирования ингредиента (все поля обязательны, кроме `author_id`).
- Кнопка удаления — с учётом прав и проверкой использования.

## 9. Планировщик меню

### Общее

- Доступен только **авторизованным** пользователям.
- У каждого пользователя **один** план меню на неделю (пн–вс × завтрак/обед/ужин = 21 слот).
- План сохраняется **автоматически** при каждом изменении (drag-and-drop, изменение порций, удаление блюда из слота).
- Кнопка «Очистить всё» — сбрасывает все слоты.

### Структура страницы

**Верхняя часть — панель выбора рецептов:**

- Горизонтальный список рецептов (2–3 ряда карточек), достаточный для выбора.
- Строка поиска/фильтрации рецептов.
- Переключатели режима: «Все» / «Избранное» / «Мои» (рецепты, где автор = текущий пользователь).
- Карточка рецепта в панели показывает: миниатюра фото + название.

**Нижняя часть — сетка меню:**

- Сетка 7 колонок (пн–вс) × 3 строки (завтрак / обед / ужин).
- Адаптивная, но не уменьшается ниже минимального размера при отсутствии карточек; по умолчанию занимает всё доступное место.
- В каждый слот можно добавить **несколько** блюд, в том числе повторяющихся.

### Карточка блюда в слоте

- Миниатюра фото + название рецепта.
- Поле количества порций (по умолчанию — `servings` из рецепта).
- Кнопка «Удалить» — убирает блюдо из слота.

### Drag-and-drop

- Пользователь перетаскивает карточку из панели выбора в нужный слот сетки.

### Связь со списком покупок

- Кнопка «Перейти к списку покупок» — переход на страницу списка покупок, где ингредиенты уже агрегированы по текущему плану меню.
- Список покупок обновляется **автоматически** при любом изменении плана меню.

## 10. Список покупок

### Общее

- Доступен только **авторизованным** пользователям.
- Список формируется автоматически из текущего плана меню пользователя.
- Если план меню пустой — список покупок пустой.
- Обновляется **автоматически** при любом изменении плана меню.

### Агрегация ингредиентов

- Одинаковые ингредиенты из разных блюд суммируются в одну строку.
- Количество пересчитывается с учётом порций, заданных в слотах планировщика.
- Единица измерения берётся из ингредиента (не переопределяется).

### Отображение

- Таблица с группировкой по **категориям ингредиентов**.
- Внутри каждой группы — сортировка по алфавиту.
- Колонки: название ингредиента, суммарное количество, единица измерения.

### Действия

- **Скопировать в буфер обмена** — копирует список текстом (название, количество, единица).
- **Распечатать** — открывает диалог печати браузера (`window.print()`).

## 12. Навигация и общая структура

### Главное меню (шапка)

Пункты меню в шапке (порядок слева направо):

1. **Рецепты** — список рецептов с поиском
2. **Планировщик** — планировщик меню на неделю (только авторизованным)
3. **Список покупок** — список покупок (только авторизованным)
4. **Ингредиенты** — каталог ингредиентов
5. **Категории** — управление категориями (только администратору; решение о совмещении с «Ингредиентами» — на стадии дизайна)
6. **Дашборд** — статистика

> Расположение «Категорий» и «Ингредиентов» в навигации уточняется на стадии дизайна (возможно объединение в один раздел).

### Авторизация в шапке

- В правой части шапки отображается текущий пользователь (имя / аватар-заглушка).
- Кнопки «Войти» / «Зарегистрироваться» — для неавторизованных.
- Кнопка «Выйти» — для авторизованных.

### Пагинация

- Применяется к: списку рецептов, каталогу ингредиентов.
- Размер страницы по умолчанию: **100 элементов**.
- Виджеты дашборда (топ-10 и т.д.) пагинации не имеют — всегда фиксированное количество.
