# NFR-0003: Frontend (Next.js + BFF)

Нефункциональные требования к frontend-сервису `apps/web` (UI + BFF).
См. ADR-0015..0019, AR-0009 (frontend TS/Node), AR-0010 (BFF boundary).

## Производительность (UI, средняя мобила, 4G)

| Метрика | Цель |
|---|---|
| LCP p75 | ≤ 2.5 s |
| TTI p75 | ≤ 3.0 s |
| INP p75 | ≤ 200 ms |
| CLS | ≤ 0.1 |
| Initial JS на страницу | ≤ 200 KB gzip |
| Lighthouse Performance | ≥ 80 |

## BFF (серверная часть frontend)

| Метрика | Цель |
|---|---|
| Latency overhead BFF (без backend) | p95 ≤ 30 ms |
| Latency агрегированной страницы (BFF + backend) | p95 ≤ 150 ms, p99 ≤ 300 ms |
| Throughput пик (MVP) | 50 RPS |
| Throughput горизонт 12 мес | 200 RPS |

## Доступность

| Метрика | Цель |
|---|---|
| Uptime web-сервиса | 99.9 % / месяц (≈43 мин даунтайма) |

## Восстановление

| Метрика | Цель |
|---|---|
| RPO | 0 (BFF stateless, signed encrypted cookie; конфиг в git) |
| RTO | ≤ 5 мин (рестарт контейнера) |

## Безопасность

- JWT никогда не попадает в браузер (см. AR-0014 / frontend-домена эквивалент).
- Session cookie: `httpOnly` + `Secure` + `SameSite=Lax`.
- CSRF-защита для всех мутаций (Server Actions и POST/PUT/DELETE на BFF).
- В браузерном бандле отсутствуют raw JWT и серверные секреты (проверка в CI).

| Метрика | Цель |
|---|---|
| Время старта контейнера `web` до readiness | ≤ 10 с |
| Логи | stdout контейнера (`docker logs`) |

## Compliance

Не применяется (учебный проект, без PII-обработки).

## Сценарии и use cases

- UC1: открытие карточки рецепта (SSR через BFF, один агрегированный запрос).
- UC2: поиск/фильтрация рецептов (SPA-навигация после первой загрузки).
- UC3: drag-and-drop планировщика меню на неделю.

## Out of scope (MVP)

- PWA / offline-режим.
- Визуальная регрессия.
- Многоязычность (i18n).
- A/B-тесты, аналитика, телеметрия.
- CDN перед edge proxy.
- Хранилище изображений (S3-совместимое) — отдельный ADR в будущем.
