# Workflow: add-report

## Алгоритм

1. Прочитать `REPORT.md` (если существует).
2. Если файл не существует — создать с заголовком `# REPORT` и разделами:
   - `## AI-инструменты`
   - `## Проблемы и решения`
   - `## Удачные шаги`
   - `## Неудачные шаги`
   - `## Примеры промтов`
3. Определить целевой раздел по описанию пользователя:
   - Упоминание инструмента / модели / AI → `## AI-инструменты`
   - Упоминание проблемы / ошибки / препятствия → `## Проблемы и решения`
   - Упоминание успеха / хорошего результата → `## Удачные шаги`
   - Упоминание неудачи / переработки / отката → `## Неудачные шаги`
   - Упоминание промта / примера / запроса → `## Примеры промтов`
   - Если раздел неоднозначен — спросить пользователя.
4. Получить текущую дату в формате `YYYY-MM-DD`.
5. Добавить запись в конец целевого раздела:

   ```markdown
   ### <Краткий заголовок> (YYYY-MM-DD)

   <Описание события, проблемы, решения или примера>.
   ```

6. Сохранить файл.

## Правила

- Язык записей: русский.
- Формат: Markdown, соблюдать [стандарт оформления](../../docs/standards/markdown-code-style.md).
- Не удалять и не переписывать существующие записи.
- Дата — текущая дата в формате `YYYY-MM-DD`.
