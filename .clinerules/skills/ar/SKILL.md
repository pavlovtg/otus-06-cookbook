Перед началом — прочитать `ARCHITECTURE.md`.

## Определение домена

- Конкретный сервис → имя сервиса (например, `order-service`).
- Несколько сервисов → bounded context, application или `backend`/`frontend`.
- Технология → имя технологии (`dotnet`, `postgresql`).
- Общие правила → `general`.
- При сомнении — спросить у пользователя.

Домен определяет только путь к файлу (`docs/architecture/rules/<domain>/...`). В шапке AR поле «Домен» не указывается. Статус по умолчанию — активное правило, в шапке не указывается.

## Нумерация

- Монотонная в рамках всего репозитория, не повторяется.
- Следующий номер = max существующий + 1 (проверить все папки `docs/architecture/rules/`).

## Создание AR

1. Определить домен и следующий номер.
2. Создать файл `docs/architecture/rules/<domain>/AR-NNNN-<kebab-name>.md` по шаблону `assets/ar-template.md`.
3. Применить шаг «Консистентность».

## Удаление AR

Когда AR больше не актуально:

1. Удалить файл `docs/architecture/rules/<domain>/AR-NNNN-<kebab-name>.md`.
2. Применить шаг «Консистентность».

## Запреты на дубли

- НЕ дублировать в файле краткое описание из `ARCHITECTURE.md`.
- НЕ добавлять блоки `## Назначение`, `## Когда применяется`, `## Обоснование`, `## Разрешено`, `## Связанные ADR`, `## Связанные документы`.
- Единственная допустимая ссылка — одна строка `Источник: ADR-NNNN[, ADR-MMMM]` сразу после заголовка (опционально). Других ссылок в файле нет.
- Шапка ограничена шаблоном. Поля `Домен` и `Статус` не добавлять.

## Консистентность

1. Обновить индекс AR в `ARCHITECTURE.md` (одна строка-описание — единственный источник краткого описания).
2. При наличии явного ADR-источника — указать строку `Источник: ADR-NNNN`.
3. Перекрёстные ссылки в ADR/standards/диаграммах НЕ создаются и НЕ обновляются — граф централизован в `ARCHITECTURE.md`.
4. Обновить `docs/architecture/overview.md`/`components.md` ТОЛЬКО если правило влияет на структуру/компоненты.

## Принципы

- Каждый AR — небольшой, независимый, легко читаемый документ.
- AR — источник истины для архитектурных ограничений.
- Устаревшие AR удаляются (не архивируются).
