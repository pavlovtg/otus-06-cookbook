Перед началом — прочитать `ARCHITECTURE.md`.

## Инструменты

- Все диаграммы создаются на **PlantUML**.
- Для архитектурных диаграмм использовать **C4 Model** (C4-PlantUML).

## Типы C4

- **Context** — системный контекст (C4Context).
- **Container** — контейнеры системы (C4Container).
- **Component** — компоненты контейнера (C4Component).
- **Dynamic** — динамические взаимодействия (C4Dynamic).

## Путь

`docs/architecture/diagrams/<kebab-name>.md`.

## Создание диаграммы

1. Создать файл `docs/architecture/diagrams/<kebab-name>.md` по шаблону `assets/diagram-template.md`.
2. Применить шаг «Консистентность».

## Обновление диаграммы

1. Обновить файл диаграммы.
2. Применить шаг «Консистентность».

## Консистентность

При любом изменении диаграммы:

1. Добавить/обновить ссылку на диаграмму в подходящий документ:
   - `ARCHITECTURE.md` — если диаграмма важна для навигации.
   - `docs/architecture/overview.md` — если описывает общую структуру.
   - `docs/architecture/components.md` — если описывает компоненты или зависимости.
   - Конкретный ADR/AR/стандарт — если относится к нему.
2. Проверить актуальность всех существующих ссылок на диаграмму.
3. При сомнении, куда добавить ссылку — спросить у пользователя.
