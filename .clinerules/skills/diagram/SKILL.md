## Инструменты

- Все диаграммы — на **PlantUML**.
- Архитектурные — **C4 Model** (C4-PlantUML): Context / Container / Component / Dynamic.

## Путь

`docs/architecture/diagrams/<kebab-name>.md`

## Создание / обновление

1. Создать или отредактировать файл диаграммы по шаблону `assets/diagram-template.md`.
2. Применить шаг «Консистентность».

## Запреты на дубли

- НЕ дублировать краткое описание из `ARCHITECTURE.md`.
- НЕ добавлять блоки `## Связанные компоненты` / `## Связанные ADR` / `## Связанные AR` и их варианты.
- НЕ добавлять секцию `## Тип` — тип C4 виден из кода диаграммы.
- Ссылки на ADR/AR — только одной строкой `Источник: ADR-NNNN[, AR-MMMM]` сразу после заголовка.
- Обратные ссылки из ADR/AR/standards/overview/components на диаграмму НЕ создаются — навигация через индекс в `ARCHITECTURE.md`.

## Консистентность

1. Обновить индекс диаграмм в `ARCHITECTURE.md` (одна строка-описание — единственный источник краткого описания).
2. Перекрёстные ссылки в ADR/AR/standards НЕ создаются и НЕ обновляются.
3. Обновить `docs/architecture/overview.md` / `components.md` ТОЛЬКО если диаграмма меняет описанную в них структуру (без добавления ссылки на саму диаграмму).
