Перед началом — прочитать `ARCHITECTURE.md`.

## Определение домена

- Конкретный сервис → имя сервиса (например, `order-service`).
- Несколько сервисов → bounded context, application или `backend`/`frontend`.
- Технология → имя технологии (`dotnet`, `postgresql`).
- Общие решения → `general`.
- При сомнении — спросить у пользователя.

Домен определяет только путь к файлу (`docs/adr/<domain>/...`). В шапке ADR поле «Домен» не указывается.

## Нумерация

- Монотонная в рамках всего репозитория, не повторяется.
- Следующий номер = max существующий + 1 (проверить все папки `docs/adr/`, включая `archive/`).

## Создание ADR

1. Определить домен и следующий номер.
2. Создать файл `docs/adr/<domain>/ADR-NNNN-<kebab-name>.md` по шаблону `assets/adr-template.md`.
3. Применить шаг «Консистентность».

## Архивирование ADR

Когда ADR устаревает или заменяется новым:

1. Обновить поле `**Статус**` в файле ADR: `устарел` или `заменён ADR-NNNN`.
2. Переместить файл из `docs/adr/<domain>/` в `docs/adr/archive/`.
3. Применить шаг «Консистентность».

## Запреты на дубли

- НЕ дублировать в файле краткое описание из `ARCHITECTURE.md`.
- НЕ добавлять блок `## Связанные документы`, `## Связанные AR`, `## Связанные стандарты` и любые их варианты.
- ADR не ссылается на AR и standards. Допустимы только ссылки на другие ADR (например, в поле «Заменяет» или внутри `## Контекст` при объяснении предыстории).
- Блок `## Рассмотренные варианты` не выделяется отдельной секцией — альтернативы упоминаются одной строкой внутри `## Контекст`.
- Шапка ограничена шаблоном: только `Статус`, `Дата` и опц. `Заменяет`. Поле `Домен` не добавлять.

## Консистентность

1. Обновить индекс ADR в `ARCHITECTURE.md` (одна строка-описание — единственный источник краткого описания).
2. Если ADR заменяет предыдущий — установить поле `Заменяет` и обновить статус прежнего ADR.
3. Перекрёстные ссылки в AR/standards/диаграммах НЕ создаются и НЕ обновляются — граф централизован в `ARCHITECTURE.md`.
4. Обновить `docs/architecture/overview.md`/`components.md` ТОЛЬКО если решение влияет на структуру/компоненты.

## Иммутабельность

ADR иммутабельны после принятия. Для изменения решения — создать новый ADR с полем `**Заменяет**`.
