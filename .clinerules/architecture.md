# Архитектура

## Правила

- В начале каждой задачи — прочитать `ARCHITECTURE.md` и следовать политике загрузки контекста.
- При создании/редактировании Markdown — соблюдать `docs/standards/markdown-code-style.md`.
- При любой операции с ADR — использовать skill `adr`.
- При любой операции с AR — использовать skill `ar`.
- При любой операции со стандартом — использовать skill `standard`.
- При любой операции с диаграммой — использовать skill `diagram`.

## Формат документов и граф ссылок

- Краткое описание (одна фраза для агента) для каждого AR/ADR/standard ведётся ТОЛЬКО в `ARCHITECTURE.md` — внутри файлов не дублируется.
- ADR: секции `Контекст` / `Решение` / `Последствия`; immutable; альтернативы — одной строкой внутри `Контекст`. Без блока «Связанные документы».
- AR: секции `Правило` / (опц.) `Запрещено`; опц. строка `Источник: ADR-NNNN` сразу после заголовка. Без `Назначение`/`Когда применяется`/`Обоснование`/`Разрешено`/`Связанные ADR`.
- Standard: секция `Правила`; опц. строка `Источник: ADR-NNNN`. Без блока «Связанные AR и ADR».
- Шапка минимальна: домен = путь файла; статус AR и ADR по умолчанию (`активно`/`принят`) не указывается; дата — только в ADR.
- Граф ссылок направленный и плоский:
  - ADR → ∅ (только на другие ADR в полях «Заменяет» или внутри `Контекст`);
  - AR → опц. `Источник: ADR-NNNN[, ADR-MMMM]`;
  - Standard → опц. `Источник: ADR-NNNN[, ADR-MMMM]`.
- Перекрёстные ссылки AR↔AR, AR↔standard, обратные AR/standard→ADR в виде блока «Связанные» — запрещены. Навигация — через `ARCHITECTURE.md`.
